let & const

  • 首先 var 会有变量提升的概念, 而 let 和 const 没有, 其次 let 声明的是变量, 而且只能声明一次, 再次声明会报错, const 声明的是一个常量, 定义完后不可以再次修改, 如果想要修改改成一个对象即可, 两者都为一个块级作用域

    注意: const 声明变量使用大写变量, 并且声明时必须赋值

  • for 循环时 let 和 var 的区别

    1. 使用 var 时, 整个程序运行到结束用的都是同一个循环变量的值; 而使用 let 时，每次循环时，let 定义变量的结果都做了独立模块的解析
    2. 在处理异步的时候, 会在循环结束后再执行, var 声明的是个全局变量, 所以等到循环结束后 var 声明的变量就是最后的结果, 而 let 所声明的变量在每次循环中都是独立的

模版字符串

以前是通过 变量 + '常量' 来处理, 而模版字符串是通过反引号 `常量${变量}` 来处理, 相当于增强版的字符串, 除了可以增加变量之外, 还可以定义多行字符串

箭头函数

Arrow Function, 相当于匿名函数, 简化了函数行定义, 没有 this、arguments、super、new.target

注意:

  1. 箭头函数的 this 指向为函数定义时所在的作用域(函数定义时所在的作用域), 而普通函数的 this 指向的是绑定该事件的元素(this 等于触发事件的函数)
  2. 箭头函数不能作为构造函数使用, 因此, 使用 new、prototype 等操作时会报错

函数的参数默认值

在 ES6 中, 可以给参数设置默认值, 当该参数没有被定义时, 默认值会生效

解构赋值

从一个数组/对象/字符串/数值/布尔/函数参数中获取其中一些属性

  • 如果解构不成功, 会返回 undefined
  • 如果解构类型有错误, 会直接报错

  2. 对象的解构

对象超类(super方法)

super 关键字表示原型对象时，只能用在对象的方法之中，用在其他地方都会报错

扩展运算符

主要用于函数调用, 将一个数组变为参数序列, 其内部使用的是 for ... of 循环

相比 ES5 将数组转为函数的参数时不需要再使用 apply 方法

应用:

  1. 复制数组
  2. 合并数组
  3. 可以和解构赋值结合
  4. 字符串: 可以将字符串转为真正的数组
  5. 实现了 Iterator 接口的对象
  6. Map 和 Set 结构, Generator 函数
