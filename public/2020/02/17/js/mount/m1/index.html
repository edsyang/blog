闭包

对函数内私有变量的一种保护机制

  • 专业解释: 函数执行形成私有作用域, 在私有作用域中声明变量不受外界影响, 这种机制叫做闭包
  • 常⻅解释: 形成⼀个不销毁的作用域, 才是闭包
  • 闭包的两大作用: 保护 和 保存

利用闭包实现高级编程技巧

  1. 柯里化函数（Currying）: 把多参数的函数变成单参数的函数
  2. 利用闭包机制隔离全局命名空间: 在过去没有模块规范的时候⼤家通常用这种方式区分命名空间
  3. 惰性封装
  4. 利用闭包不销毁作用域保存数据

闭包经典问题

JS 编译机制: VO/AO/GO

  • VO: Variable Object
  • AO: Activation Object
  • GO: Global Object

JS 引擎执行过程分为三个阶段

  1. 语法分析
  2. 预编译阶段
  3. 执行阶段

    注: 浏览器首先按顺序加载由 <script> 标签分割的 JS 代码块, 加载 JS 代码块完毕后, 立即进入以上三个阶段, 然后再按顺序查找下一个代码块, 再继续执行以上三个阶段, 无论是外部脚本文件(不异步加载)还是内部脚本代码块, 都是一样的原理且都在同一个全局作用域中

1. 语法分析

分析该 JS 脚本代码块的语法是否正确, 如果出现不正确, 则向外抛出一个语法错误, 停止该 JS 代码块的执行, 然后继续查找并加载下一个代码块, 如果语法正确, 则进入预编译阶段

2. 预编译阶段

JS 的运行环境以下三种

  • 全局环境: JS 代码加载完毕后, 进入代码预编译即进入全局环境
  • 函数环境: 函数调用执行时, 进入该函数环境, 同步的函数则函数环境不同
  • eval: 不建议使用, 会有安全、性能等问题
    每进入一个不同的运行环境都会创建一个相应的执行上下文(Execution Context), 那么在一段 JS 程序中一般都会创建多个执行上下文, JS 引擎会以栈的方式对这些上下文进行处理, 形成函数调用栈(call stack), 栈底永远是全局执行上下文, 栈顶则永远是当前执行上下文

函数调用栈

函数调用栈就是使用栈存取的方式进行管理运行环境, 特点是先进后出, 后进先出

创建执行上下文

执行上下文可理解为当前的执行环境和该运行环境相对应; 创建执行上下文的过程中, 主要做了三件事:

  1. 创建变量对象(VO)

    「创建变量对象主要经过以下过程」

    1. 创建 arguments 对象: 检查当前上下文中的参数, 建立该对象的属性和属性值, 仅在函数环境(非箭头函数)中进行, 全局环境没有此过程
    2. 检查 function 函数声明创建属性: 按代码顺序查找, 将找到的函数提前声明; 如果当前上下文的变量对象没有该函数名属性, 则在该变量对象以函数名建立一个属性, 属性值则为指向该函数所在堆内存地址的引用, 如果存在则会被新的引用覆盖
    3. 检查 var 变量声明创建属性: 按代码顺序查找, 将找到的变量提前声明; 如果当前上下文的变量对象没有该变量名属性, 则在该变量对象以变量名建立一个属性, 属性值为 undefined, 如果存在则忽略该变量声明

      注1: 在全局环境中, window 对象就是全局执行上下文的变量对象, 所有的变量和函数都是 window 对象的属性方法, 所以函数声明提前和变量声明提升是在创建变量对象中进行的, 而函数声明优先级高于变量声明

      注2: 创建变量对象发生在预编译阶段, 但尚未进入执行阶段, 该变量对象都是不能访问的, 因为此时的变量对象中变量属性尚未赋值, 值仍未 undefined, 只有进入执行阶段, 变量对象中的变量属性进行赋值后, 变量对象(VO) 转为活动对象(AO) 后, 才能进行访问, 这个过程就是 VO 到 AO 的过程

  2. 建立作用域链(Scope Chain)
    作用域链由当期执行环境的变量对象(未进入执行阶段前)与上层环境的一些列活动对象组成, 它保证了当前执行环境对符合访问权限的变量和函数的有序访问

作用域（栈内存 stack）

给 js 提供运行环境、存储基本数据类型

  • 全局作用域: 打开页面时形成, 用来执行全局中的代码, 就是 window 对象（Node 中的 Global）; 全局变量（不带变量的键值对, 如 a = 1）会给 window 加一个键值对, 全局都可以直接使用（window. 可以省略）
  • 私有(函数)作用域: 函数执行时形成的作用域; 外界获取不到
  • 块级作用域: 代码块（if、for、let、const等）变成作用域; 注意: 对象的 {} 是一种数据类型格式而不是作用域（因为不能执行代码, 不存在域的概念）

作用域链

变量的查找机制; 使用变量时, 浏览器会在当前作用域中查找, 如果找到就使用私有变量（形参、私有变量）, 如果找不到, 向上级作用域查找, 直到 window 为止, 如果 window 还没有, 此时有两个情况:

  1. 取值（访问一个变量）: 报错;
  2. 赋值: 给 window 添加一个属性

    如果定义上级作用域: 看这个函数在哪个作用域定义的, 定义时所在的作用域, 就是该函数的上级作⽤域

堆栈内存

js 引擎中内存分类（浏览器在执行代码时会形成两个虚拟的内存）

  1. 堆内存: 存储引用数据类型的值
  2. 栈内存（作用域）: 提供 js 代码执行的环境, 存储基本数据类型的值

    堆栈内存常见问题

    • 函数执行时开辟的栈内存是在全局的栈内存开辟的, 它会消耗全局作用域的空间, 函数的作用域特别多的话会影响代码执行的效率;
    • 堆内存特别大也会影响浏览器器执⾏的效率

堆内存的创建和释放

  • 创建堆内存: 创建⼀个引用数据类型的值(对象、数组、正则、日期、函数), 浏览器会(默认)为这个引用数据类型创建一块堆内存, 这个堆内存就是存这些引⽤数据类型的值;
  • 释放堆内存: 将所有引用该堆内存的变量或者属性全部置为 null(要等浏览器空闲的时候才会去垃圾回收, 释放这些没有被引用的内存), 浏览器也会检测, 当引用数据类型不被外界所占用时, 将所占空间回收

栈内存的创建和释放

  • 创建栈内存
    1. 浏览器打开一个⻚面时, 会⾸先形成⼀个顶层的栈内存, 就是全局作用域 window;
    2. 函数执行时产生的私有作用域, 就是一个供函数代码执的栈内存「注意: 仅仅创建一个函数是不会形成一个新的栈内存, 函数执行的时候才会有栈内存」
  • 释放栈内存
    1. 全局作用域销毁: 页面关闭时销毁
    2. 普通函数: 函数执行完后(没有返回值 / 返回值是基本数据类型的值 / 返回值是引用数据类型的值但不被外界接收和占用), 立刻销毁
    3. 不销毁作用域
      • 当函数执行完, 如果函数返回了一个引用数据类型的值, 并且被外面的变量或者属性接收, 此时函数执行的作用域不能销毁, 在这个作⽤域中储存的值也不会销毁;
      • 函数内部的引用数据类型值, 被函数外的变量或者属性占用, 此时函数作用域不能销毁
    4. 不立即销毁作用域: 函数执行的返回值是个函数, 外层函数执行时得到里面的函数再执行, 这样外层函数执行完后不会立即销毁, 而是会等到里面的函数行完后再销毁

this

js 的关键字, 代表当前代码执行的环境对象; 一般在函数中使用并且是在函数指定时, 根据函数的不同执行方式确定不同的值「注意: this 在运行时不能赋值」

this 的几种情况

  1. 全局作⽤域下, this 指向 window: this 和 window 的空间地址是一样的, window 下如果没有某一个属性, undefined
  2. 事件函数中的 this 指向绑定该事件的元素(this等于触发事件的函数)
  3. 方法调用(函数执行)时, 看方法前⾯是否有点（.）如果有, 点前面是谁, ⽅法中的 this 就指向谁, 如果没有, ⽅法中的 this 就指向 window, 如果有多个点, 只看它执⾏前面的那⼀个即可
  4. 自执行函数中的 this 指向 window
  5. 定时器 setTimeout() / setInterval() 回调函数中的 this 指向 window
  6. 构造函数 (constructor) 中的 this 指向实例（原型上的方法中 this 不一定指向实例, 还需要看这个方法是谁调用的）
  7. 箭头函数中 this 指向函数定义是所在作用域（上级作用域）中的 this「注意: 箭头函数中没有 arguments, 但是可以使用不定参数」
  8. call、apply、bind 可以改变 this 指向（这三种方法都在 Function.prototype 内; 第一个参数如果是 null、undefined 或者不传参的情况下 this 仍指向 window; use strict 下第一个参数是 null、undefined, this 就指向 null、undefined）
    • call(): 修改函数中 this 的指向, 并且把修改 this 后的函数执行; 函数名.call(ctx, 实参1, 实参2...) ctx 就是用来替换函数中 this 的对象(修改 this 指向, 新的 this 指向 ctx), 从第二个参数起, 都是传递给函数的实参
    • apply(): 修改函数中 this 的指向, 并且把修改 this 后的函数执行; 函数名.apply(ctx, [实参1, 实参2...]) ctx 就是用来替换函数中 this 的对象(修改 this 指向, 新的 this 指向 ctx), 第二个参数是以数组的方式打包传递给函数的实参
    • bind(): 修改函数中的 this, 返回一个修改 this 后的新函数（新函数执行才会改变 this 指向）, 不会让函数执行（预处理 this, 用法和 call 一样）; 函数名.bind(ctx, 实参1, 实参2...) ctx 就是用来替换函数中 this 的对象(修改 this 指向, 新的 this 指向 ctx), 从第二个参数起, 都是传递给函数的实参

Function.prototype.bind = function (context) {
  if (typeof this !== 'function') {
    throw new Error('Function.prototype.bind - what is trying to be bound is not callable');
  }
  // 获取调用 bind 的函数
  var self = this;
  // 获取 bind 函数从第二个参数到最后一个参数
  var args = Array.prototype.slice.call(arguments, 1);
  var FNOP = function () {};
  var fbound = function () {
    // 当作为构造函数时, this 指向实例, self 指向绑定函数
    // 当作为普通函数时, this 指向 window, self 指向绑定函数
    self.apply(this instanceof self ? this : context,
      args.concat(Array.prototype.slice.call(arguments)));
  }
  // 修改返回函数的 prototype 为绑定函数的 prototype, 实例就可以继承函数的原型中的值
  // 通过空函数中转, 防止直接修改函数的 prototype
  FNOP.prototype = this.prototype;
  fbound.prototype = new FNOP();
  return fbound;
}
