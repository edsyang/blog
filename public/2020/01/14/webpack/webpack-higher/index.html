删除 Webpack 项目中的无用文件

当项目发展一段时间后, 会产生很多无用文件; 可能删除了某个路由, 但是没有删除代码, 也可能是删除了某个页面, 但是没有删掉引用的组件

原理: Webpack 是从一个入口文件 entry 开始, 逐层递归往里走, 找到所有文件的, 所有有用的文件会组成一个以入口文件为开始的树, 这棵树以外的文件都是没用的文件

  1. 安装依赖 npm i unused-files-webpack-plugin --save-dev
  2. 修改 config/config.js, 添加插件
  3. 执行 npm run dev, 这时候可以在日志中看到无用文件的列表
  4. 拷贝列表到一个 txt 文件中, 假设叫 unused.txt
  5. 通过 node.js 脚本来读取 txt, 并删除响应文件
  6. 文件删除后, 会产生很多空文件, 递归删除空文件
