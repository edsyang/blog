package.json 中的版本前缀

  • 波浪符号 ~: 把当前库的版本更新到当前 minor version(中间的数字)中最新的版本
  • 插入符号 ^: 把当前库的版本更新到当前 major version(第一位数字)中最新的版本

WIP

work in progress -> doing/todo

PR

pull request

当你想更正别人仓库里的错误时, 要走一个流程：

  1. 先 fork 别人的仓库, 相当于拷贝一份
  2. clone 到本地分支, 做一些 bug fix
  3. 发起 pull request 给原仓库, 让他看到你修改的 bug
  4. 原仓库 review 这个 bug, 如果是正确的话, 就会 merge 到他自己的项目中

至此, 整个 pull request 的过程就结束了

SDK / API / DLL

  • SDK(software development kit) 软件开发工具包
  • API(application programming interface) 应用程序编程接口
  • DDL(Dynamic Link Library) 动态链接库

SDK

SDK 一般是指软件工程师为特定的软件包、软件框架、硬件平台、操作系统等建立应用程序是开发的合集(相关文档、规范、工具、API); 通俗来讲就是第三方服务商提供的实现产品某项功能的工具包

  • 场景一: 老板让做一个中后台管理系统, 但是我觉得时间太长了, 不利于开发上线, 偶然间看到了某大厂平台上有相关功能, 于是我可以在基于平台的 SDK 下, 调用该平台的 API, 就可以访问平台的资源和数据
  • 场景二: 需要做一个 APP, 其中包含6个功能, 首先要做1-6的 API, 组成 SDK, 然后根据 SDK 开发 Project File 项目文件, 制作 APP, 如果后续需要加新的功能, 将新功能的 API 做好后, 组成新的 SDL, 生成新的 Firmware

还有一个粗暴的解释:

你是一男的, 要开发一个 baby, SDK 就是房间、床、老婆等一系列相关的工具、文档; API 就是老婆开放的接口; API 固然重要, 但如果想有一个优质的体验, 就需要将 SDK 做好

Schema

数据库对象的集合, 这个集合包含了各种对象如: 表、视图、存储过程、索引等; 为了区分不同的集合, 就需要给不同的集合起不同的名字, 默认情况下一个用户对应一个集合, 用户的 schema 名等于用户名

如果把 database 看作是一个仓库, 仓库很多房间（schema）, 一个 schema 代表一个房间, table 可以看作是每个房间中的储物柜, user 是每个 schema 的主人, 有操作数据库中每个房间的权利, 就是说每个数据库映射的 user 有每个schema（房间）的钥匙

CI/CD

  • CI (Continuous Integration) 持续集成
    • 当开发每天会提交多次代码到主干上, 会做一些重复性的动作时, 就可以用持续集成环境来操作
  • CD (Continuous Deployment) 持续部署
    • CD 还有一个小号, 也叫 CD (Continuous delivery)

CI/CD 的优点是重复的工作用自动化来代替、减少时间成本、版本发布时间减短了

Jenkins 就是一个持续集成的工具, 基于 java 语言的开源工具

IaaS & PaaS & SaaS

  1. IaaS (Infrastructure as a Service), 基础设施即服务; 把计算基础(服务器、网络技术、存储和数据中心空间)作为一项服务提供给客户, 它也包括提供操作系统和虚拟化技术、来管理资源, 消费者通过 Internet 可以从完善的计算机基础设施获得服务

    云服务的最底层, 主要提供一些基础资源, 用户需要自己控制底层, 实现基础设施的使用逻辑; 如 Amazon EC2、Digital Ocean、RackSpace Cloud

  2. PaaS (Platform as a Service), 平台即服务; 把服务器平台作为一种服务提供的商业模式, 一个作为软件开发和运行环境的整套解决方案, 即以 SaaS 的模式提交给用户, 因此, PaaS 也是 SaaS 模式的一种应用, 但是, PaaS 的出现可以加快 SaaS 的发展，尤其是加快 SaaS 应用的开发速度

    提供软件部署平台(runtime), 抽象掉硬件和操作系统细节, 可以无缝扩展(scaling), 开发者只需要关注自己的业务逻辑, 不需要关注底层; 如 Heroku、Google App Engine、OpenShift

  3. SaaS (Software as a Service), 软件即服务; 一种交付模式, 其中应用作为一项服务托管, 通过 Internet 提供给用户, 帮助客户更好地管理它们的 IT 项目和服务、确保它们 IT 应用的质量和性能, 监控它们的在线业务

    软件的开发、管理、部署都交给第三方, 不用关心技术问题, 拿来即用; 如客户管理服务、团队协同服务、储存服务、存储服务、社交服务

举个栗子

假设一个人想做披萨生意, 可以从头到尾自己生产, 但是这样比较麻烦, 需要准备的东西比较多, 因此决定外包一部分工作, 采用他人的服务, 现在有三个方案:

  1. 「IaaS」他人提供厨房、煤气、炉子, 你使用这些基础设施来烤披萨
  2. 「PaaS」除了基础设施, 他人还提供披萨饼皮, 只要把自己的配料放到饼皮上, 让他帮你烤出来就行了; 也就是说你要做的就是设计披萨的味道, 他人提供平台服务, 让你把自己的设计实现
  3. 「SaaS」他人直接做好披萨, 不用你介入, 到手的就是一个成品, 你要做的就是把它卖出去, 最多在包装一下, 印上自己的 Logo
