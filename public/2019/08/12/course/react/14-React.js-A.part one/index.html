0. 前言

学习框架最重要的就是要清楚各种组件间的对应关系;

React 是数据驱动的；当数据（state、props）发生变化的时候，React 的组件会重新执行 render 函数进而更新视图

1. 初识React

React是由Facebook开源的前端框架，核心是专注于视图，目的是实现组件化开发

官方原文档：https://reactjs.org

官方中文文档：https://react.docschina.org

  • 组件化的概念

可以很直观的将有一个复杂的页面分割成若干个独立的组件，每个组件包含自己的html结构、样式和js逻辑，然后再将这些组件组合，即可完成一个复杂的页面。这样既减少了逻辑的复杂度，又实现了代码的重用；

  • 组件化的优势
  1. 可组合：一个组件可以和其他组件一起使用或者可以直接嵌套在另一个组件内部
  2. 可重用：每个组件都具有独立的功能，他可以被使用在多个场景中
  3. 可维护：每个小的组件仅仅包含自身的逻辑，更容易被理解和维护

3. React的组成部分

React由两部分组成：

  1. react.js【React的核心库】
  2. react-dom.js【提供与React DOM相关的能力，常用的方法：reactDOM.render(节点, document.getElementById('root'), callback（将节点渲染到页面后所执行的操作）) 这个方法可以把react元素（其实就是jsx元素）渲染到页面真实的DOM元素中】

4. JSX语法

JSX语法是React特有的语法，是一种 js 和 html 混写的语法，将组件的 html 结构、数据、样式写在一个 js 文件中（聚合在一起定义组件），最终都会经过编译形成普通的 js 代码

JSX是一种语法糖，目的是简化 html 和数据绑定的过程，提升开发效率

  • JSX元素（react元素）：在 jsx 中长得像 html 元素的就是 jsx 元素
  • JSX语法可以写在 js 文件中，也可以写在 .jsx 文件中，但是如果要写在 js 文件中，需要先导入 react

4.1 使用JSX的注意事项：

  1. JSX 通过 ReactDOM.render() 渲染到页面中，JSX 元素最外层只能有一个根节点
  2. JSX 本质是 js，不是 html，写 JSX 要遵守所有的 js 语法；例如：return是关键字，不能作为变量名
  3. JSX 中可以通过 {} 使用 js 表达式，注意只能写表达式（变量、三元运算符、方法的调用（函数执行）、数学表达式），不能写语句；例如
  4. JSX 元素可以使用行内属性，但是有些特殊的属性需要改写：class 是 js 关键字，需要改写成 className；lable 标签 for 改写成 htmlFor ；style 属性需要写成对象形式

    for 属性规定 label 与哪个表单元素绑定，如果在 label 元素内点击文本，就会触发此控件。就是说，当用户渲染该标签时，浏览器就会自动将焦点转到和标签相关的表单控件上。

  5. JSX 元素（react元素）并不是真正的元素（DOM），在渲染到页面之前都是虚拟 DOM ，不能当作真实的 DOM 来操作；JSX 经过 babel 编译会形成普通的 js 对象，这些对象就是虚拟 DOM
  6. JSX 如果换行，最好用 () 包起来

4.2 ReactDOM.render() 渲染

ReactDOM.render(ele, container, callback)

  • ele: 要渲染到页面的 jsx 元素或者组件
  • container: 要把上面的 ele 渲染到哪个元素中，是一个 DOM 对象
  • callback: 把 ele 渲染到页面后执行的回调函数，在这个回调函数中可以操作 DOM（如果需要操作刚刚渲染到页面中的元素，在这里操作）
  1. 在执行 render 的时候，把 jsx 的虚拟 DOM 转换成真实的 DOM 元素
  2. 把上一步 jsx 转换得到的真实的 DOM 元素插入到页面中已经存在的 DOM 元素中
  3. 调用 render 的回调函数

4.3 在JSX中使用CSS样式

在 React 中使用样式，把 css 文件通过 import 导入即可；最后打包的时候 webpack 会自动处理 css

不能省略 .css 扩展名；jsx 中不能用 class 为元素指定类名，要用 className

5. 列表渲染&条件渲染

React 中没有类似 Vue 那样的指令系统，但是 react 有列表渲染和条件渲染

  • 列表渲染：使用数组的 map 方法
  • 条件渲染：使用 if-else 等判断语句或者表达式

5.1 列表渲染

列表渲染：根据给定的数据，生成一组相同的 DOM 元素；在 Vue 中 v-for 指令做列表渲染；而在 React 中使用数组的 map 方法做列表渲染，在 map 的回调函数中返回（return）要生成多个的 jsx 元素（map 把一个数据映射成一个新数组），注意：列表渲染的时候需要写 key

当使用列表渲染出来的结果时，只需要把结果当成普通变量，用 {} 包裹一下；这样写有一个前提：不使用列表渲染，HTML结构该写成什么样子，只有先清楚这个问题才能用列表渲染写出来

培养一种数据映射视图的能力，看到 HTML 就要想到数据结构应该长成什么样，看到数据结构时，要想到它渲染成 HTML 会长成什么样子

以下方法常配合列表渲染使用

  • Object.keys() 把对象的属性名取出来组成一个数组
  • Object.values() 把对象的属性值取出来组成一个数组
  • Object.entries() 把对象的属性名和属性值组成一个新数组，形如：[ [‘key1’, ‘value1’], [‘key2’, ‘value2’] … ]

5.2 条件渲染

条件渲染：根据条件成立与否，生成不同的 html 结构；React的条件渲染，需要使用 if-else 等判断语句或者表达式

5.3 练习：列表渲染

6. React.createElement

React.createElement() 方法，用来创建虚拟 DOM，而 jsx 语法中，jsx 元素就是虚拟 DOM，jsx 语法就是 createElement 的语法糖

React.createElement(type, props, ...children)

  • type：最外层标签名
  • props：type 的行内属性；是一个对象，如果没有行内属性，写 null
  • children：从第三个参数起，就是 type 的子节点，如果其子节点还有元素，则继续调用 createElement 方法

7. React的组件

使用 react 写一个每秒更新一次的时钟

React 和 Vue 都是数据驱动的，但是上面的时钟不是数据驱动的，而是每秒生成一个 jsx，然后渲染到页面中；如果要使用数据驱动，就需要使用 React 的组件

React 支持组件化开发，在 React 组件中，jsx 元素（也称 react 元素）是基本组成单位，在 React 中有两种定义组件的方式

  1. 函数（function）定义组件
  2. 类（class）定义组件

定义组件需要注意：

  • 1.组件的名字首字母必须大写，为了在写 jsx 时有别于原生的 HTML 标签
  • 2.在函数中要返回一个根 jsx 元素
  • 3.定义组件以后，这个组件就可以当成 HTML 标签使用了

7.1 React - function（函数）组件

用函数声明组件，就是创造一个函数

  • 函数接收一个形参 props，props 是一个对象(数据类型的)，其中包括在渲染该组件或者父组件在使用该组件的时候（组件在被使用的时候通过其行内属性的方式传递过来的数据），在该组件行内传入的 props（属性/对象）
  • 函数需要返回一个根 jsx 元素，在组件中需要的数据可以通过 props 传入 ReactDOM.render() 会根据第一个参数的不同，采取不同的操作
  1. 如果第一个参数是一个组件，首先会把组件的行内属性进行打包封装，将其打包成一个对象传给组件函数
  2. 执行组件函数并且传递上一步打包出来的 props 对象，获取组件函数返回的虚拟 DOM 对象
  3. 把虚拟 DOM 转成真实的 DOM，并插入到页面（真实的 DOM）中

7.2 React - class（类）组件

通过 class 定义/声明一个组件：

  1. 通过 class 来定义一个组件，需要继承 React 上的 Component 这个类
  2. 在定义组件（类）的原型上必须有一个 render 方法（函数），并且 render 函数需要返回一个顶级的（根） jsx 元素
  3. 用 class 声明的组件，声明后就可以当作标签使用

    写在 render 方法 return 后面的 jsx 相当于 .vue 文件中的 template 标签里面的内容

  • 用 class 声明的组件，同样可以使用 props，但是在构造函数中，构造函数的第一个形参就是 props 对象，在构造函数中通过形参访问 props，除了构造函数以外的其他地方使用，通过 this.props 访问 props ReactDOM.render() 渲染 class 声明组件的过程
  1. 找到组件对应的类，然后 new 一下这个类，获取这个类的实例
  2. 通过实例找到类原型上的 render 方法，执行 render 方法并接收其返回的虚拟 DOM
  3. 把上一步的虚拟 DOM 转换成真实的 DOM，然后插入到页面中

7.3 class&function定义的组件的不同

React 也是数据驱动的，当数据（props、state）发生变化时，视图就会自动发生变化（视图是数据的映射），组件中的数据是有两个来源：props（属性） 和 state（状态），其中 props 就是组件被使用时接收的行内属性，是从外部传入的数据，而 state 是组件的私有数据，组件定义时就需要创建；

用 class 声明的组件，有 state、props、this、生命周期的钩子函数；而 function 声明的组件在没有使用 Hooks 的时候，只有 props

Hooks：React 16.8 新增特性，可以在不编写 class 的情况下使用 state 等其他 React 特性

  • props 是在组件被使用的时候写在其行内的属性，是从外部传给组件的数据
  • state 是用 class 声明的组件私有的数据，需要定义组件时就要声明状态

    props 和 state 都是组件的数据来源，如果 props 和 state 发生变化，对应的视图会自动更新

8. 数据映射视图

把组件中的数据（props、state）绑定到 jsx 中，进而渲染到页面中

8.1 属性（props）映射视图

bouns: 使用别人的组件时，一般都会有文档，文档会告诉传哪些属性，如果用就要按照文档传属性

属性（props）也是组件的数据，而视图是数据的映射，当数据发生变化时，组件会自动重新渲染

8.2 状态（state）映射视图

React 组件的数据有两个来源：props 和 state
属性（props）：是父组件或者渲染时传递过来的数据，是外部传入的
状态（state）: 是组件自己管控的状态，状态是组件私有的数据，是组件内部的

使用state

在 React 中在不使用 Hooks 的情况下，如果使用 state 必须使用 class 创建组件

  • 使用 state 需要在 constructor 中初始化 state

  • 初始化 state 就是给 this.state 赋值成一个对象

  • 注意：state 里面的状态（数据）不可以直接修改，如果需要更新/修改 state 状态数据，需要调用 setState() 方法，setState 有两种用法：

    • 1.函数形式：this.setState((prevState) => {state: newValue}) prevState 就是被修改之前的状态；setState 的回调函数需要返回一个对象，对象中的属性就是要变更的状态，不需要变更的状态不需要写在对象中，React 会自动合并
    • 2.对象形式：this.setState({state: newValue})

  • state 发生改变后触发 render 函数执行更新 DOM

    tips：使用普通函数，如果不做特殊处理，this 并不是当前组件的实例（可以使用bind）；而事件函数声明为一个箭头函数，函数中的 this 就是当前组件的实例；切记不能直接修改 state 的状态（数据）

    当我们修改状态后，我们发现视图自动更新了； React 是数据驱动的，而 state 是它的数据，我们更新数据后，React 会自动重新调用组件的 render 方法更新视图；

    当 React 发现数据变化了，需要更新视图；React 会重新调用组件的 render 重新获取虚拟 DOM，然后和原有的视图进行对比，只更新变更的那一部分视图，其他的不更新；（DOM-diff 算法）

  • 在 render 方法中，可以把 state 中的状态绑定到 jsx 中；this.state.xxx 的方式可以获取 state 中的 xxx 状态

  • 在react中绑定事件：react 绑定事件时，需要使用小驼峰命名法的事件名：onClick = {事件处理函数}

9. 父子组件通信

9.1 父传子

在 React 中，父组件向子组件传递数据通过 props（跟 Vue 一样）

  1. 父组件的 state 可以通过 props 传递给子组件
  2. 父组件的 props 也可以通过 props 传递给子组件
    Panel.js Header.js

9.2 子传父

在 React 中，子组件修改父组件数据的方式和 Vue 不同

  1. 当父组件使用子组件的时候，通过 props 把一个可以修改父组件数据的方法传递给子组件
  2. 当子组件想要修改父组件的数据时，子组件通过 this.props 找到父组件传过来的方法，执行它就可以修改父组件的数据了

10. props（属性）校验

React 的 props 也支持校验，但是需要一个第三方的库：prop-types

  1. 安装 prop-types
  2. 导入 prop-types
  3. 在类中声明 propTypes 的静态属性，值是一个对象，对象的 key 就是需要被校验的 props， value 是校验规则；使用时，如果组件收到的 prop 不符合规则，会触发控制台警告

11. 受控组件&非受控组件

  • 受控组件，受 React 的 state 控制的组件，表单的数据存在 state 中，需要数据时直接在 this.state 中获得即可
  • 非受控组件的数据仍然保存在 DOM 元素上，如果想要获取这个元素的 value，可以通过操作 DOM 的方式获取

11.1 受控组件

因为 React 是单向数据绑定，所以需要使用受控组件来实现双向数据绑定

受控组件：在 HTML 中，input、textarea、select 等表单元素，可以根据用户的输入改变自身的状态（就是表单的 value）；而 React 中可变状态存在 state 中；把表单的状态和 state 中的数据结合起来，让 React 的 state 成为单一数据源，这样的组件称为受控组件

通俗来讲，就是表单元素的 value 属性绑定组件 state 中的数据（状态），当表单的数据发生变化（onChange）时，在事件函数中更新 state 中的状态；即受控组件就是受 state 控制的组件

注意：如果要使用受控组件，一定要设置 onChange 事件更新 state 中的状态

11.2 非受控组件

非受控组件：不受 React 的状态控制的组件，非受控组件的 value 存在 DOM 元素上（表单元素的 value 不再绑定 state 中的状态），此时需要使用 ref 获取表单元素的值（需要操作 DOM）

  • 在 React 中操作 DOM 需要使用 ref，使用 ref 就是在你想获取的元素上增加 ref 属性，值就是构造函数创建的 ref

使用 ref 的步骤

勿过度使用 Refs，让更高的组件层级拥有这个 state 是更恰当的

  1. 在构造函数中创建 ref
  2. 把上面创建的 ref 赋值给想要获取的元素的 ref 属性
  3. 如果需要获取执行 ref 的 DOM 对象，ref 上的 current 属性就是带有指定 ref 的 DOM 元素对象，操作它跟操作普通 DOM 元素对象一样
  • ref 的第二种使用方式：给 ref 赋值成一个箭头函数，this.myRef 就是该 DOM 对象的 ref，该函数有一个形参 el，el 表示使用当前 ref 的 DOM 对象，这个函数会在元素挂载后执行，执行后在父组件的 this 上的 myRef 就是这个 DOM 元素了（不用再使用 current，通过 this.myRef 可以直接获取）

12. React的生命周期

只有 class 声明的组件才有生命周期，constructor 和 render 都是生命周期中的钩子函数

componentDidMount、componentWillUnmount 基本上是最常用的钩子函数

  • 组件在初始化时执行的钩子的顺序：static defaultProps（这个是静态资源，不是钩子函数） -> constructor -> componentWillMount -> render -> componentDidMount

  • 组件在数据更新时执行的钩子的顺序：shouldComponentUpdate -> componentWillUpdate -> render -> (子组件数据更新钩子) -> componentDidUpdate

    当父组件的数据发生变化时，执行到 render 时才会触发子组件关于数据更新时的钩子函数

    React 16.9 Updata

    componentWillMount → UNSAFE_componentWillMount
    componentWillReceiveProps → UNSAFE_componentWillReceiveProps
    componentWillUpdate → UNSAFE_componentWillUpdate
