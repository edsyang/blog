14. Redux

Counter.js

Computed.js

在 React 中数据通信，父子间通过 props 传递，如果数据很多会很不方便，而且非父子组件间无能为力；为了优化过程，我们使用 redux 来管理应用的状态

14.1 使用redux

redux 是一种数据管理模式，它不仅可以用于 react 还可以配合其他库和框架使用；把整个应用的状态交给 redux 托管，redux 会导出一个 store，这个 store 中包含获取数据的方法，以及变更数据的方法

当然，数据和更新数据的方法都需要自己提前定义好

使用步骤：

  1. 在组件中导入 store
  2. 用 store 中的状态初始化组件中的 state，例如：
  3. 把状态映射到视图上，就是通过 this.state.属性名 绑定到 jsx 元素上
  4. 当需要修改状态的时候调用：store.dispatch(action对象)，action 对象中要有 type 字段以及传递给 reducer 的 payload
  5. 如果要更新视图，需要在 componentDidMount 中订阅状态更新
  6. 取消订阅需要在 componentWillUnmount 中调用函数
1. 安装redux
2. 创建store
  1. 创建 store 需要使用 redux 的 createStore 方法，使用前需要先导入
  2. createStore 创建 store 需要告诉 store 托管的数据以及修改这些数据的方法（告知 store 托管的数据和修改数据方法通过一种叫做 reducer 的函数完成），就是将 reducer 函数传给 createStore

    被 redux 托管的状态（数据）不能直接修改状态，需要定义修改状态的函数，这个函数称为 reducer

reducer 函数都会接收两个参数（注意：写 reducer 函数首先要先 return state）：

  • state：当前 redux 托管的数据（状态），state 在 reducer 函数中的默认值就是 state 的初始值
  • action：修改状态具体的动作以及修改状态需要的参数（payload 载荷）；action是一个带有 type 属性的对象，而 reducer 就是根据不同的 action.type 返回一个新的状态对象，以此实现修改状态的目的

    reducer 是修改状态的方法，但是不是用来直接执行的，如果需要修改状态，需要 dispatch(actionObj)，传给 dispatch 的对象叫做 action，action 是一个必须带有 type 字段的对象，这个 action 将来会传给 reducer 函数

store.js

3. 在组件中导入store

store 可以做：

  • store.getState() 初始化 state
  • 当数据修改时需要派发行动 dispatch action；例如：store.dispatch({type: ‘ADD’, amount: 12})，dispatch 执行时传递的对象就是 action，action 对象会传递给 reducer 函数的第二个参数
  • 如果需要数据修改后更新视图，需要订阅这个数据发生变化后的事件，如果想修改视图就更新 state，在组件挂载的钩子（componentDidMount）中订阅
  • 订阅后会有取消订阅的需要，订阅函数会返回取消订阅的函数，在组件将要销毁的钩子（componentWillUnmount）中执行取消订阅的操作
  1. 导入 store
  2. 使用 store.getState() 初始化 state
  3. 通过 store.dispatch(actionObj) 修改状态
  4. store.subscribe(callback) store 订阅状态更新后执行的回调，以便更新视图

14.3 reducer合并

redux 是单一状态树，整个应用中只能有一个 store，但是有一个组件就有一个 reducer，而 createStore 只能接收一个 reducer，此时需要合并这些 reducer，合并 reducer 需要一个 combineReducers() 方法，这个方法在 redux 上，导入即可

使用 combineReducers

  1. combineReducers 接收一个对象，对象中的属性名将会成为对应被托管状态的命名空间，如果获取状态需要通过命名空间获取：store.getState().命名空间
  2. combineReducers 会返回一个整合后的 reducer

    使用 combineReducers 整合的 store 带有命名空间，在组件中使用的时候就需要通过对应的命名空间来获取状态

15. react-redux

redux 虽然实现了数据的统一管理，但是代码组织很繁琐，为了解决这个问题我们使用一个另一个工具：react-redux

react-redux 可以简化 redux 使用过程

15.1 使用react-redux

使用步骤：

  1. 通过 Provider 组件把 store 引入组件树
  2. 用 connect 改造组件，mapStateToProps、mapDispatchToProps 以及二者的简化写法
  3. 导出 connect 连接后的组件
1. 安装react-redux
2. 使用Provider组件

react-redux 提供了 Provider 组件，可以把 store 引入到组件树中，可以简化在组件里初始化状态

  • 在应用的顶层/主入口（一般就是index.js）使用 Provider 组件，并且为 Provider 组件设置 store 属性，值就是 redux 导出的 store
3. 使用connect改造组件并导出

react-redux 的 connect 方法，connect 是一个高阶函数（组件），例如：let connect = (ms, md) => (component) => {}；connect 后面可以跟两个函数执行：通过 connect 改造组件，经过 connect 改造，组件中的数据以及派发数据的方法全部通过 props 来实现；

connect( mapStateToProps, mapDispatchToProps )( Component )

  • 导入 connect 方法
  • 使用 connect 导出的是连接后的组件，connect 可以执行两次；
    • 第一次执行传入两个回调函数：mapStateToProps、mapDispatchToProps
      • mapStateToProps 把 store 的状态映射成为组件的 props
      • mapDispatchToProps 把 dispatch 映射为 prop
    • 第二次执行传入需要被连接的组件名
  • 注意：使用 connect 后，对应组件内的使用数据和修改数据的方式也需要调整，方法和数据都要从 prop 上获取

    mapStateToProps 可以简写成一个箭头函数；mapDispatchToProps 可以简写成 actionCreator 对象

    Bouns：

    1. 先看看是不是自己的代码写的有问题，如果没有问题的话看一下步骤
    2. 如果项目跑不起来，删除 node_modules，重新安装依赖后再重启
    3. 如果安装依赖的过程中报错，清理 nom 缓存：npm cache clean --force
    4. 如果还不行，再把报错粘贴到搜索上查找报错解决方案

16. Redux中间件

中间件是用来增强 redux 功能的

使用中间件需要在 store 的 index.js（整合后的store） 中导入，并且需要从 redux 中导入 applyMiddleware() 方法

常用的 redux 中间件：redux-logger、redux-thunk、redux-promise

中间件会使用即可，暂时不必要纠结原理

16.1 redux-logger

当 store 中的状态发生变化时，redux-logger 会把状态改变之前的值和改变之后的值输出到控制台（利于调试bug）

  1. 安装 redux-logger
  2. 使用中间件：需要在 createStore 的时候使用中间件；使用中间件需要使用 redux 上的一个 applyMiddleware() 方法
  3. 在控制台修改状态，控制台会输出修改前/后的状态

16.2 redux-thunk

有时候需要一些异步的操作（ajax、定时器…），这些异步的操作无法直接在 actionCreator 中进行；使用 redux-thunk 中间件可以使 actionCreator 返回一个函数；这个函数有两个参数：dispatch 和 getState；dispatch 就是 store.dispatch，getState 就是 store.getState；相当于 dispatch 的权利交给了这个（actionCreator 返回的）函数，在这个函数中，想什么时候更新状态（dispatch）就什么时候更新状态，所以就可以在定时器和 ajax 中更新 store 中的状态了

  1. 安装 redux-thunk
  2. 使用中间件
  3. 使用后的 actionCreator 函数

16.3 redux-promise

在日常开发中，尤其是处理异步的数据时，常用到 Promise；对此 redux-promise 中间件专门用来处理 redux 中的 Promise；

  • 使用 redux-promise 后，actionCreator 中使用 Promise 有两种用法：
    • 1.直接在 actionCreator 中返回一个 Promise，这种用法只能处理 resolve 的情况，并且是在 resolve() 传入 action 对象
    • 2.如果 resolve 和 reject 都要处理，此时需要在 actionCreator 中返回一个 action 对象，这个 action 对象有一个 payload 属性，payload 的属性值是一个 Promise 实例；如果 Promise resolve，payload 属性的值就是 resolve() 执行传入的值；如果 Promise reject，payload 就是 reject() 执行传入的值；如果 reducer 用到了 resolve 或者 reject 的值，对应的 reducer 也需要改
  1. 安装 redux-promise
  2. 使用中间件
  3. 使用后的 actionCreator 函数
