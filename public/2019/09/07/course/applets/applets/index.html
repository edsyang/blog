创建文件的方式：

  1. 直接在app.json中的pages加文件名，如”pages/log/log”，command/ctrl+s保存后该文件夹会自动出现在pages文件夹中
  2. 或者直接在pages内新建一个目录，如log，在log目录下新建一个log.js，其他的会自动生成

rpx：微信自己封装的px，以iphone6屏幕大小为基础，1rpx = 0.5px，会根据屏幕大小自适应

事件前缀

  • bind支持冒泡事件
  • catch不支持冒泡事件
    事件写法
  • 事件前缀+事件类型，如bindtap
  • 事件前缀:事件类型，如bind:tap

路由

  • wx.navigateTo：保留当前页面，打开一个新页面（非tabBar页面）
  • wx.redirectTo：关闭当前页面，打开一个新页面（非tabBar页面）
  • wx.reLaunch：关闭所有页面，打开一个新页面（所有类型类面）
  • wx.switchTab：打开一个tabBar页面，关闭非tabBar页面（如果是tabBar页面，那么就保留当前页面）
  • wx.navigateBack：返回n级页面，前提是要有那么多

生命周期

  • onLoad 监听页面加载
  • onReady 监听页面初次渲染完成
  • onShow 监听页面显示
  • onHide 监听页面隐藏
  • onUnload 监听页面卸载

行内属性

  • wx:for=’‘ 循环
  • wx:for-item=’’ 给item重新命名
  • wx:key=’‘
  • wx:for-index=’’ 给index重新命名
  • wx:if 控制组件的有无，会是组件销毁和重新显示，造成页面组件的回流
  • wx:else
  • hidden=’‘ 控制样式，不会引起页面组件的回流，在频繁切换组件隐藏显示的时候可以使用

页面配置文件

  1. index.js 他就是当前页面的逻辑文件，主要存放页面逻辑和一些动态的属性
  2. index.json 他就是当前页面的配置文件
  3. index.wxml 他存放的就是当前页面的数据展示，相当于html层
  4. index.wxs 他就是当前页面的样式文件

在iphone6下1rpx就等于0.5px ，所以咱们拿到的小程序设计稿都是以6为准的
类选择器 标签选择器 子级选择器 并集选择器 伪类选择器

事件

  • 冒泡事件 bindtap
  • 非冒泡事件 catchtap
  • 事件的写法 1、bindtap bind:tap
  • 事件源： e.currentTarget.dataset

生命周期和路由
一. 路由

  1. navigateTo 保留当前页面，打开一个非tabbar页面（当前页onHide执行 新页面的onLoad、onShow、onReady）
  2. redirectTo 关闭当前页面 打开一个非tarbar页面（当前页面的onUnload执行 新页面的onLoad、onShow、onRead）
  3. reLaunch 关闭所有页面 打开一个新页面 （当前打开所有页面的onUnload全部执行 新页面的onLoad、onShow、onRead）
  4. switchTab 关闭非tabbar页面，打开一个tabbar页面 （如果当前页面是tabbar页面，那么就是onHide执行 ，如果不是，那就是onUnload执）
  5. navigateBack 回退的前提是前面得有被打开的页面 （当前页面onUnload执行 返回的页面onShow执行）

页面传参

  1. 通过路由传参：在路径后面直接拼参数
  2. 跳转之前获取getApp()，把要传的参数设置到app.js中

跳转成功之后获取getApp()，然后拿到你之前设置的参数

组件

  1. 定义组件：新建components
    1. 使用组件：在父页面的json文件里的usingComponents里以key、value引入一下，key就是引入的组件名称，value就是组件路径
    2. 以标签的形式直接在父页面的wxml里直接使用即可
  2. 组件父传子：
    1. 在父页面的使用子组件的标签的行内直接定义一个属性，如name=’123’，在子组件的properties内定义一下类型name: String，也可以name:{type: String, value: ‘000’}
    2. 在父页面的使用子组件的标签的行内定义data-name=’123’，在子组件里的this.dataset直接可以取到这个值
  3. 子传父：
    1. 在子组件定义this.triggerEvent(‘事件名’, ‘事件参数’)
    2. 在父页面使用子组件的标签的行内 bind事件名=’函数名’，bind的事件名必须要和子组件的triggerEvent的事件名一样
    3. 接着在父页面中给这个函数名定义
