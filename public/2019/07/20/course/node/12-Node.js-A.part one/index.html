1. Node.js基础

https://Node.js.org/ 基于C++开发的，语法是JavaScript;

window + r  ->  cmd  ->  Node -v （查看Node的版本号）
             ->  npm -v （查看npm版本号）

1.1 什么是Node.js

Node.js不是一门新的语言，而是基于Chrome V8引擎的，一个可以让JS能够运行在服务端的运行环境（runtime），也可以理解成是一个虚拟机（VM）；在服务器上安装Node.js以后，就可以用JS开发服务端的程序，所以JS现在是全栈开发语言。

开发一个完整的项目，需要前后端相互配合；前后端的代码都是保存在服务器上的；前端的代码运行在浏览器上，而后端的代码运行在服务器上；

前后端通过http协议通信，前端通过http协议把请求发送给服务器，服务器接收客户端的请求准备客户端需要的数据，最后通过http协议把数据响应给客户端

服务端的作用

  1. 存储前后端的代码（JS、HTML、CSS、img等文件）
  2. 处理客户端的请求
  3. 响应客户端的请求（把数据发送给客户端）
  4. 安装数据库，保存用户数据

传统的服务端语言

  • Java
  • PHP
  • C C++（可以和Node.js合作）
  • .net
  • python
  • Golang（Go语言，可以和Node.js合作）

    新贵：Node.js

Node.js的优势

Node.js可以写任何最新的技术、ES6/ES7语法任意用，没有兼容性问题
  1. 单线程异步，事件驱动
  2. 事件驱动的非阻塞I/O（读写），轻量高效 【采用异步的方式处理I/O（Input/Output）】
  3. npm：Node Package Manager 包管理器工具；npmjs.com 全球最大的Node开源生态系统
  4. npm是随着Node一起安装的，安装完Node，npm也就随着安装完成
  5. npm包就是做好的库，只需要安装后再导入即可使用
  • 安装nrm：npm install nrm -g
    • nrm作用：切换源（源就是从哪下载）
      • nrm ls 【查看所有的源，*就是当前所在源】
      • nrm use taobao 【切换至taobao源】
  • 安装yarn：npm install yarn -g
    • yarn也是Node.js的包管理器，yarn的特点就是速度快；
    • yarn 等效于npm install；按照package.json安装依赖
    • yarn add 依赖包名 –save 【安装生产依赖】
    • yarn add 依赖包名 –dev 【安装开发依赖】
    • yarn remove 依赖包名 【写在依赖包，等效于npm uninstall 依赖包名】

      指定版本号：（如果不指定版本号默认最新版本）
      npm install 依赖包名@版本号 –xxxx
      yarn add 依赖包名@版本号 –xxxx

如果安装报错先清缓存：npm cache clean –force

2. Node.js的模块

  • 在Node.js中，没有HTML，JS之间引用就需要模块化
  • 在Node.js中，所有的JS文件都是模块；所有的方法和逻辑都要写在模块内

Node的模块分为三种

  1. 内置的核心模块：安装Node时，随着Node一起安装的模块，这些模块都是Node.js的核心模块；如http模块、url模块、fs模块等
  2. 自定义模块：自己写的模块
  3. 第三方模块：通过npm/yarn安装的第三方模块；如nrm、yarn、lessc等

    无论哪种模块在使用之前都需要导入；导入内置模块和第三方模块时都不需要些路径；导入自定义模块时必须写路径

在Node.js中运行JS

  1. 找到要运行的JS文件所在目录
  2. 在JS文件所在的目录中，按住shift+右键，选择在此处打开PowerShell窗口（win7 在此处打开命令窗口）/ 在当前JS文件所在的目录搜索栏执行cmd，跳出命令提示符
  3. 在命令行界面中输入：node 文件名 -> 回车
  • 在vscode/webstorm中使用terminal（终端运行）：在项目中找到JS所在目录的文件夹 -> 右键Open In Terminal -> 执行Node 文件名 -> 回车
  • 插件运行：
    • vscode：安装code runner插件 -> 右键run
    • webstorm：在JS文件中右键选择 run文件名

2.1 模块的使用

因为Node.js没有html，如果JS文件互相依赖，就需要使用模块化把这个JS文件中的内容导出，需要把这些内容的模块导入

所谓模块化就是把别人需要的东西导出去，如果不导出，其他人无法使用；把自己需要的东西导入进来；别人的模块导出什么，我们导入时就能拿到什么，别人没导出的，我们拿不到；解决多个JS互相依赖（如：b.js要使用a.js文件中的方法或者属性，a.js中要把其他模块需要的方法或者属性到导出，b.js需要导入使用所需要的方法

导出模块方式：

  • module.exports = fn; / module.exports = xxx; 直接导出fn；一次只能导出一个，否则后面把前面的覆盖
  • exports.fn = fn; 导出一个对象，对象中有一个属性fn，属性值是fn这个函数
  • module.exports.fn = fn; 等效于 exports.fn = fn;

    如果要导出多个使用exports.属性名 = 属性值 形式；其中属性名自定义；但是你导出的时候叫什么，导入时还需要使用同名的属性

导入模块方式：

  • 使用require(‘被依赖的JS文件路径’) 方法；导入时需要用变量接收导入的结果然后再通过变量使用导入信息

    在导入js文件时，js中的代码就会执行；导入JS/JSON文件可以不写扩展名.js/.json

    • 求7和3之差，并且输出到console

2.2 模块的原理

在浏览器中，JSON、setTimeout… 都是定义在window对象中；而Node.js中没有window对象，全局对象是global

Node.js天生自带模块化，Node.js会给JS文件外面套一个自执行函数，并且给自执行函数传入几个实参：

  • exports：导出
  • module：模块
  • require：导入模块的方法
  • __dirname：当前JS文件夹所在目录的绝对路径
  • __filename：当前JS文件名，带绝对路径和扩展名

2.3 第三方模块

使用第三方的模块称为依赖包；需要使用包管理器（npm/yarn）来安装（下载）/ 管理

通过npm在没有依赖包的目录中安装依赖包的步骤：

  1. 通过npm生成一个package.json文件：npm init (-y) ；这个package.json是用来记录当前项目中的依赖包的，如框架、类库、脚手架等
    • npm init 需要自己填写内容
    • npm init -y 自动生成

      注意：文件路径不要有中文！目录中不要有汉字，文件和文件夹不要叫Node/npm/node_module

  2. 安装项目依赖包（根据项目需要安装依赖包）
  • 生产依赖和开发依赖
    • 生产依赖：项目在运行的时候需要的依赖包，如jQuery
    • 开发依赖：只在开发的过程中有用的依赖包（大多数都是工具性的包，如gulp、webpack等打包工具）
  1. 安装依赖包的命令：npm install 依赖包名称 参数
  • npm install 参数：
    • 无参数 根据package.json所记录的，安装到Node_modules中；默认当作生产依赖
    • -g 全局安装（安装成cmd的一个命令）；全局安装后在命令行里面可以使用
    • -save 安装生产依赖；把依赖包下载Node_modules下，并且把这个包的信息添加到package.json下的dependencies中
    • -save-dev 安装开发依赖；把依赖包下载Node_modules下，并且把这个包的信息添加到package.json下的devDependencies中
  • package.json
    • 记录了当前项目所需的依赖包和版本
    • dependencies 记录的当前项目所需要的生产依赖包【简单理解：就是让代码在运行的时候需要的依赖包】
    • devDependencies 记录的当前项目所需要的开发依赖包
    • 根据package.json中记录的开发以来和生产依赖全装对应的依赖包：npm install

      因为package.json文件记录了所有的依赖包，所以只需要在package.json所在的项目目录执行npm install；mac需要加上sudo 以管理员身份运行该命令：sudo npm install

  • node_modules
    • 安装依赖的时候会自动生成这样一个文件夹；所有通过npm/yarn安装的依赖包都会放在这个里面
    • 文件夹不会来回传递，如果上线并不会把Node_modules传到服务器上

      清理npm缓存（当安装依赖包失败的时候先清缓存）：npm cache clean –force

  1. 卸载安装包
    • npm uninstall 依赖包名称

2.4 Node.js的内置模块

Node.js内置模块是在Node安装时已经随Node一起安装的模块，这些模块都是Node的核心模块，如fs，http，url模块等，不用自己写，直接导入就可以使用

2.4.1 fs模块

fs模块 - 读取文件

fs模块是Node.js内置的模块；file system 文件系统；专门用于处理文件的读写

使用fs模块先导入fs模块，其他内置模块也需要在使用前导入

  • res.end() 向客户端发送响应数据，并在发送完成后断开连接
  • res.send() 向客户端发送响应数据，但发送完成后不断开链接
  1. 异步读取文件
  • fs.readFile(path, option, callback)
    • path：带路径的文件名
    • option：设置读取的内容为哪种编码，默认值是buffer，存储二进制的数据，一般在机器之间传递可以直接传二进制；（默认值可省略）
    • callback：回调函数，读取文件后无论读取成与否，异步执行回调函数
  1. 同步读取文件
  • fs.readFlieSync(path, option)
    • path：带路径的文件名
    • option：编码

      这个方法的返回值是读取到的内容，如果读取失败直接报错

      Node.js报错要查看行号的话需要先找到自己写的js文件，然后再找到行号

fs模块 - 修改文件

修改文件：向文件中写入内容

  1. 异步写入/修改文件
  • fs.writeFile(path, data, option, callback)
    • path：文件名（如果没有这个文件会创建一个）
    • data：要写入文件的内容
    • option：文件编码
    • callback：写入后执行的回调

      fs.writeFile() 是覆盖式写入，原文件中的内容会被覆盖掉；如果要追加的话，先把原来的文件内容读取出来，拼接上要写入的内容，然后再一并写回去

  1. 同步写入/修改文件
  • fs.writeFileSync(path, data, option)
    • path：带路径的文件名
    • data：写入文件的内容
    • option：编码

      同步写入没有返回值

fs模块 - 向文件中追加内容
  1. 异步追加内容
  • fs.appendFile(path, data, option, callback)
    • path：带路径的文件名
    • data：写入文件的内容
    • option：编码
    • callback：追加后的回调函数
  1. 同步追加内容
  • fs.appendFileSync(path, data, option)
    • path：带路径的文件名
    • data：写入文件的内容
    • option：编码

2.4.2 http模块

前端（客户端）：发起请求，获取数据，渲染页面
后端（服务端）：响应请求，发送数据

前后端通过HTTP协议通信，前端通过HTTP发请求，向后端请求数据；后端处理HTTP请求，通过HTTP协议向前端发送数据

HTTP模块：http是Node.js的内置模块，是用来处理客户端的http请求的

  • 写完脚本以后，还需要到命令行内启动这个服务：node 文件名
  • 去浏览器中访问：localhost:8000

    为什么用localhost？因为正常的请求是访问url，DNS负责把url解析成ip地址，然后请求ip地址；但是服务是运行在本地，IP地址的ip 127.0.0.1，而127.0.0.1 就是localhost

此时电脑上既运行着客户端（浏览器）又运行着服务端（用node启动的js脚本）

2.4.3 url模块

url模块也是Node.js的内置模块，用来解析url

当修改了服务端的程序后需要重启这个server（ctrl+c/ctrl+z），但npm上有一个supervisor第三方依赖包可以实现自动重启server

  • url.parse(req.url, true) 解析请求的url的方法
    • req.url 客户端请求的url
    • true 将url问号传参格式化成一个对象，并挂载在query属性上
    • pathname 客户端请求的路径
    • search 客户端问号传参的内容（带问号）
    • query 客户端传递的查询字符串

3. 根据请求路径不同返回不同内容

根据请求的路径不同返回不同的html

思路：获取当前的请求路径，读取路径对应的html然后返回给客户端即可

服务端响应如html、css、js、图片等文件服务称为静态资源服务；html、css、js等文件称为静态资源；返回具体文件的都是静态资源，

如果pathname 类似 /home/help/search这样的格式，这种请求路径不是一个具体的文件，一般都是ajax接口；一般都是处理动态的请求，根据客户端传递的数据返回不同的内容

4. 响应与Content-Type

  • 服务端在响应客户端的请求时，要设置响应内容的内容类型，即Content-Type；当客户端收到服务端的响应内容后，会按照Content-Type解析响应内容；
  • Content-Type 是一个响应头，服务端设置

5. mime - 检测文件类型

mime：多用途互联网邮件拓展；每种文件都有自己特有的mime类型，这个类型就是在响应的时候需要设置的内容类型

mime依赖包，是一个第三方的模块，其中包含了绝大多数的文件的mime类型；

使用第三方的依赖包：

  1. 安装mime：yarn add mime –save / npm install mime –save
  2. 引用第三方的模块：require(‘mime’)

    静态资源（html、css、js、图片等），当客户端请求静态资源文件时，提供响应这些请求的服务叫静态资源服务

6. 设置响应状态码404（server-with-status-code）

当用户请求的路径不存在的时候，返回状态码404，并返回NOT FOUND文字

7. axios - 客户端发送ajax请求

axios是基于promise封装的ajax库，用于客户端发送ajax请求；

原文档：http://www.axios-js.com/docs/

中文文档：https://www.kancloud.cn/yunye/axios/234845

安装axios：npm install axios –save

因为ESModule实现不完全，浏览器不能直接使用ESModule语法，所以需要在使用script标签之前引入到html中
CommonJS：导出：module.export.fn = fn ；导入：require(模块)
ESModule：导出：export；导入：import

7.1 axios常用的方法

GET、POST、ALL

GET请求

  • axios.get(url?key=value&key2=value2)；返回Promise实例，可以直接.then
    • 问号传参
    • 直接把参数放到一个对象中

POST请求

  • axios.post(url, data)；返回Promise实例，可以直接.then
  • url：接口 ； data：对象，post的数据

all并发多个请求，这些请求都结束之后再做处理

  • axios.all([ axios请求1, axios请求2… ])；返回Promise实例，可以直接.then，但是then方法中需要传入一个axios.spread(callback)
