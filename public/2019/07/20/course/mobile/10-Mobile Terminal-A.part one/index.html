1. 认识移动端

移动端：相对于PC端而言
常见的移动设备：智能手机、平板（ipad）、阅读器（Kindle等）...
智能手机：两大平台 IOS Android
  • 原生APP：直接运行在操作系统中的程序；
    • IOS: objective-c （oc） 开发的
    • Android: Java 开发的

      原生APP的优点：1. 性能好，流畅度高，可以不依赖于网络运行；2. 原生app把需要的资源都打包到安装包里面，一次安装终身使用；

原生APP的缺点：1. 不能跨平台运行，iOS 的app只能运行在ios设备中，Android的app只能在安卓系统运行；所以就需要开发两套，一般公司原生app开发团队有两拨人，一拨负责iOS开发，另一拨负责安卓；2. 时效性差，app一旦安装到用户的设备中，更新与不更新用户有主动权；（服务供应商最多可以采用强制升级，强制升级用户体验差）

大前端：包含原生app开发在内的所有面向用户的一端；

  • WebApp：运行在浏览器（以及app的webkit内核或者webview）中的html页面；
    • 我们讨论的移动端运行在移动设备的浏览器（或者app中的webkit内核或者webview）中的html页面；

WebApp 优势：1. 可以跨平台运行，web是运行浏览器中的，一套页面既可以在ios的浏览器中运行，也可以在安卓的浏览器运行；2. 时效性强，更新版本直接修改html页面、js文件等，修改完以后直接发布到服务器上，用户看到的就是最新的，更新的主动权在服务供应商手里。

WebApp 缺点：1. 依赖于网络，没有网络或者网络特别差的时候，无法使用（PWA应用可以有效缓解这个问题: https://segmentfault.com/a/1190000012353473?utm_source=tag-newest） ；2. 每次使用都需要重新加载，用户需要等待；

  • HybridApp：用原生app提供一个框架（webkit内核、webview），app大部分功能和逻辑用html+js写；

  • ReactNative/Ionic/Flutter/Electron(写pc桌面程序)：可以把前端的html+js打包成原生的app。

开发移动端需要掌握的东西：

1. html5的新特性：websocket（双工通信）、geolocation（获取地理位置，获取你的经纬度；除了经纬度还可以使用ip地址定位位置）
2. HTML5新增的标签
3. CSS3部分新特性(过渡动画、变形动画2d/3d)
4. 移动端的事件、移动端常用类库
5. 本地存储localStorage
6. 音视频audio、video

在移动端没有IE的兼容性问题，可以使用新技术

移动端的难点在于移动设备屏幕适配问题

bonus: 如果有需要在移动(手机)端做判断的需求: 如果是访问的www的网站,就跳到对应的m站; 例如: 在手机上访问www.jd.com就要跳到m.jd.com; 有一个插件: device.js 可以判断设备类型; https://www.npmjs.com/package/device.js ; window.location.href = ‘m.jd.com’ 跳转到指定页面

4. CSS3动画

4.1 CSS3的过渡动画 - transition

过渡效果：当属性发生变化的时候不是突然间到终点值，而是一点一点的到达终点值，给人一种平滑的感觉

transition是合成属性：

实例：transition: all 1s cubic-bezier(.06,.98,.83,.67) .5S

  • transition-property：过渡属性，默认值是all，可以单独设置某个属性
  • transition-duration：过渡时间，在这个时间内从初始值过渡到终点值
  • transition-timing-function：过渡效果，默认值是linear，还有ease-in ease-out 贝塞尔曲线 …

    贝塞尔曲线 cubic-bezier() https://cubic-bezier.com/

  • transition-delay: 等待时间，等待多长时间开始过渡动画

4.2 CSS3的变形动画 - transform

CSS3的变形动画需要设置transform的属性

常用属性：（变形属性可以组合使用）

  • transform: scale(水平方向尺寸缩放倍数, 竖直方向尺寸缩放倍数)
  • transform: translate(水平方向平移的距离x, 竖直方向平移的距离y) 水平方向的值越大越向右边走，竖直方向的值越大越往下面走
    • translateX(X) 只设置在x轴方向偏移量
    • translateY(Y) 只设置在y轴方向偏移量
  • transform: rotate(旋转角度deg) 角度是正值向顺时针转动，负值向逆时针转动
  • transform: skew(x轴的旋转角度, y轴的旋转角度) skew的x、y轴和不同的坐标系不同（设置斜切原点为左上角）

4.4 关键帧动画

由动画属性和2D变形组成

关键帧动画的语法（声明）：

1. 关键帧声明：from to
@keyframes 动画名 { 
    // 关键帧（关键帧元素所处的样式）
    from { 起始的样式 }
    to { 终点的样式 }
}

2. 关键帧声明：百分比（0% - 100% 的一个数字）
@keyframes 动画名 { 
    // 关键帧（关键帧元素所处的样式）
    0% { 0%的样式 }
    25% { 25%的样式 }
    50% { 50%的样式 }
    ...
}

关键帧动画的调用及设置（animation调用；transform-origin设置旋转中心点位置）

transform-origin：设置旋转动画的中心点位置
transform-origin: 水平方向 垂直方向;
实例：transform-origin: center top;
    - 水平方向：left center right
    - 垂直方向：top center bottom

animation：动画属性，是一个合成属性；这些值中为时间的要带单位
实例：animation: shake 2s linear 0s infinite;
    - animation-name：动画名称，@keyframes定义一个动画
    - animation-duration：动画时长
    - animation-timing-function：动画效果，默认值linear
    - animation-delay：延迟（多长时间再执行）动画 带单位s
    - animation-iteration-count：动画执行次数，默认1次，infinite无限次

4.6 3D动画

3D变换同样依赖设置transform属性，除了transform，还要设置视距（perspective: n px）和保留3d视图（transform-style : preserve-3d）

  • transform-3d(x, y, z) 设置元素在x、y、z轴上的偏移值
  • transformZ(z) 设置元素在z轴上的偏移值；在z轴方向上平移，如果为正值往屏幕外走
  • transform: rotateX(deg) 绕着X轴旋转；角度为正值，元素的上半边缘进入屏幕内
  • transform: rotateY(deg) 绕着Y轴旋转；角度为正值，元素的右半边缘进入屏幕内
  • transform: rotateZ(deg) 绕着Z轴旋转（在平面内转动）；角度为正值，元素顺时针旋转
  • perspective: 视距，眼睛到元素的距离（透视的概念），默认值是0，有了视距值以后才能有近大远小的效果；（根据视距近大远小的规律，越向屏幕内走，元素会越大）
  • transform-style: preserve-3d; 保留元素的3d视图，默认情况下浏览器不保留。

6.1 媒体查询

根据不同的媒体类型，如电视、阅读器、打印机、PC等 ；使不同的样式生效；常用于处理PC端的适配，不同尺寸的显示器设置（显示）不同的样式

6.2 flex布局

- flex布局是flexible Box的缩写，意为弹性盒模型，为盒模型提供最大的灵活性
- 使用flex布局需要指定容器为flex容器；给容器元素设置：display: flex
- 行内元素也可以使用flex布局：display: inline-flex
- 使用flex布局后，子元素的float、clear、vertical-align自动失效

6.2.1 基本概念

  • 容器：采用flex布局的元素称为flex容器（flex container）- 被设置了display: flex的元素，简称容器；
  • 项目：容器元素的所有子元素自动成为容器的成员，叫做项目（flex item）
  • 主轴和交叉轴：
    • 主轴：默认情况下水平方向是主轴（main axis）；主轴和元素的左边缘焦点称为main start，和右边缘的焦点称为main end
    • 交叉轴：垂直于主轴方向的是交叉轴（cross axis）；交叉轴和元素的上边缘焦点称为cross start，和下边缘的焦点称为cross end

      项目（子元素）默认沿着主轴方向排列，单个项目占据的主轴的空间（尺寸）叫做main size，占据交叉轴的空间（尺寸）叫做cross size

6.2.2 容器属性

  • flex-direction: 设置主轴方向；可选值
    • row（默认值）主轴为水平方向，起点在左端
    • row-reverse 主轴为水平方向，起点在右端
    • column 主轴为垂直方向，起点在上沿
    • column-reverse 主轴为垂直方向，起点在下沿
  • flex-wrap: 设置元素在一行排不下时如何换行；可选值
    • nowrap（默认值）不换行，靠压缩项目item的尺寸来放下
    • wrap 换行，第一行在上面
    • wrap-reverse 换行，第一行在下面
  • flex-flow: （合成属性）flex-direction和flex-wrap的简写

    flex-flow: row nowrap

  • justify-content: 设置项目item在主轴方向上的对齐方式，可选值
    • flex-start（默认值）主轴起点对齐
    • flex-end 主轴终点对齐
    • center 居中
    • space-between 两端对齐，项目item之间距离相同
    • space-around 每个项目item之间两侧的间距相同，所以项目间的距离是项目和边框距离的两倍
  • align-items: 设置项目item在交叉轴方向上的对齐方式，可选值
    • stretch（默认值）项目未设置高度或者高度auto，项目的高度自动充满整个容器
    • flex-start 交叉轴起点对齐
    • flex-end 交叉轴终点对齐
    • center 居中
    • baseline 项目第一行文字基线对齐

6.2.3 项目item属性

  • order 定义项目的排列顺序，数值越小越靠前，默认值为0
  • flex-grow 定义项目item放大的比例，默认值为0，表示有剩余空间也不放大；如果所有项目的flex-grow都为1且有剩余空间，则项目评分这些空间；如果有一个项目的flex-grow为2，其余的都为1，为2的比其他的分得剩余空间大
  • flex-shrink 定义项目item缩小的比例，默认值为1，表示空间不足时按比例缩小；如果有一个项目的flex-shrink为0，其余的都是1，那么空间不足时，为0的不缩小（但如果设置了flex-wrap则该条无效）
  • align-self 设置项目在交叉轴方向上的对齐方式和其他项目不同；可以覆盖容器元素的align-items属性；默认值auto，表示继承父元素的align-item属性
    • auto
    • flex-start
    • flex-end
    • center
    • baseline
    • stretch

6.3 rem布局

响应式布局的一种解决方案，目的还是实现在不同宽度的设备上元素占的比例是相同的

注：本质就是利用这个属性的特性完成自适应等比例缩放；rem等比例缩放元素属性在视口的占比，vw、vh等比例缩放该元素的所在位置

rem和px一样是个单位，px是绝对单位，而rem是个相对单位，rem是相对于html标签的字体大小，html标签的font-size就是1rem；html元素默认的font-size是12px，也是浏览器所能显示的最小字体（不做特殊处理）；

rem响应式的步骤：

  1. meta name=viewport -> 设置视口
  2. html { font-size: 100px } -> 设置html的fontSize
  3. rem.js -> 根据屏幕（视口）的宽动态的设置html的fontSize，因为1rem是一个html的fontSize大小，根据视口的宽动态设置html的font-size，就能达到页面整体缩放的目的

为什么使用rem布局(优点): 

如果这个页面用px来写,页面中的元素尺寸都是写死的,改成用rem来写,rem是一个相对单位,用rem设定元素的尺寸,只要修改html的font-size, rem设定的尺寸都会自动跟着缩放,而且缩放比例相同; 使用px按照某一个手机的屏幕尺寸切图, 但是换到另一个手机上, 这两个手机的屏幕尺寸不同,原来的样式就不能用了; 但是换成rem来设定元素尺寸, 当切换手机后,根据手机的屏幕尺寸(视口的宽度)修改html的font-size,页面中所有使用rem的元素尺寸都会自动按照最新的比例进行缩放

rem布局的思路:

  1. 给html的font-size设置为100px,此时页面中的1rem就是100px

  2. 写样式的时候,把所有带px的单位都改用rem来设定(测量出来的px/100 就是需要的rem值),但如果html的font-size不变,那和直接写px没什么区别;但是如果修改了html的font-size,就是修改了rem和px的换算比例,浏览器就会按照最新的比例把页面中使用rem的元素进行缩放,使用rem布局,就可以通过简单的设置html的font-size来实现整个页面的缩放

    真实项目中: 设计师会提供一套设计稿,设计稿常用尺寸的宽为640px(320px)和750px(375px),拿到设计稿后,**严格按照设计稿中的尺寸编写样式(切图)**; 首先设置html { font-size: 100px }; 此时1rem就是100px; 接下来用rem设定页面中元素的尺寸

如: html{font-size=100px} 原来在750的设备上有一个1rem的盒子, 现在运行在375的设备上, 那么当前这个盒子的宽就应该为 x / 375 = 100 / 750  =>  x = (375 / 750) * 100; 此时页面中1rem现在是50px

6.4 vw、vh布局

将视口的宽高各分为100份；每一份宽就是1vw，每一份高就是1vh；vw和vh是两个单位；在不同宽度的设备上1vw代表的px数是不同的

作用：

  1. 所有样式的尺寸都可以用vw来写，但是因为算出来的值大概率不是整数，存在四舍五入的情况，最后容易导致样式差一点，所以一般情况下还是用vw结合rem来用

  2. 让一个盒子充满屏幕：把这个盒子的宽设置为100vw，高设置为100vh

    1. 在750px的设备上1px等于多少vw？ 1/750px = x/100 -> x = 0.13333vw -> 1px等于0.13333vw -> 1vw等于7.5px
2. 在375px的设备上1px等于多少vw？ 1/375px = x/100 -> x = 0.26667vw -> 1px等于0.26667vw -> 1vw等于3.75px
