1. git基础操作

Git 和 github.com

  • git 版本管理工具，
  • github 项目代码托管平台
    - 1.注册一个github账户
- 2.新建一个仓库（remote）远程仓库

  • 2.进入到本地一个文件夹，右键 git bash here
  • 3.在弹出的命令行界面中输入 git clone 仓库地址 回车

    本地仓库（我们克隆下来的仓库）
    .git 文件夹 每一个git仓库都会有一个 不要随意删除

查看本地仓库和哪个远程仓库关联

  • 1.进入本地仓库（文件夹）
  • 2.在文件夹下右键 git bash here
  • 3.输入 git remote -v
    origin https://htt....git (fetch)

origin https://htt....git (push)

将本地文件push（推）到远程仓库

  • 提交代码需要经历git的三个区
    • 工作区：本地文件夹
    • 暂存区：缓存区域
    • 历史区：针对每次提交生成一个版本记录

提交一个版本：

    1. git status 查看仓库状态
      如果文件名都是红色的，就说明这些文件处于工作区
    1. git add . 将文件添加到暂存区
    1. git status
      文件都是绿色的，说明现在处于暂存区

    1. git commit -m "本次提交的备注" 把文件添加到历史区

      如果出现 Please tell me who you are
git config --global user.email "你的邮箱地址"
git config --global user.name "用户名"
再次执行 4. 步

      查看设置：git config -l

    1. git push 到远程仓库

      git push origin master

      输入用户名密码

    1. git log (--pretty=online) 查看提交记录

      回到指定版本：

      git reset –hard HEAD^ （^表示上一个版本）（只能将工作区修改，用最新的版本库覆盖）
      git reset –hard HEAD~n （n表示上n个版本）
      git reset –hard 版本号(id)（id没必要写全）

把本地文件夹和远程仓库关联起来

    1. git init 在文件夹下执行，把文件夹变成一个本地git仓库
    1. git remote add origin 远程仓库地址
    1. git remote -v 查看本地仓库和远程仓库的关联关系
      完成上面步骤完成本地仓库和远程关联

2. 变量提升（预解释、预处理）

代码解析的一个阶段

  1. 在js代码执行之前，浏览器会对所有带var和function的进行提前的声明或者定义；
  2. 声明（declaration）：声明一个变量，告诉浏览器有这么一个变量；
  3. 定义（define）：给变量赋值；

对带var的变量只声明不定义，变量声明后不定义其默认值是undefined；（只有名字没有值）

对带function的变量（函数名），声明并且定义；（既有名字又有值）

完成变量提升后，js代码才会从上到下执行。

  • 在执行到 var 变量 = 值；才会对带var的变量进行赋值操作，也就是说经过这一行之后变量才代表这个具体的值，在此之前变量的值是undefined。
  • 在执行到 function 函数名() {….}；浏览器会忽略这些函数定义代码，因为在变量提升阶段已经对函数名定义过了，所以不会重复定义。
  • 所以，在普通变量赋值之前使用就是undefined，赋值之后才是具体值；而function声明的函数名在声明前使用时函数本身，在声明后使用还是函数本身。

    Tips：定义的变量是变量提升到全局，但执行代码是一个标签一个标签进行的；如果提前声明，在后面的标签中也可以使用提前声明的变量；但如果在前面的标签获取不到后面标签内所定义的变量

2.1 js的运行机制

js代码执行分为两个阶段：

    1. 变量提升：js代码之前对所有带var和带function的进行提前声明或者定义；其中带var的只声明不定义，变量值是默认值undefined；对带function的既声明又定义；
    1. js的执行阶段；

js的运行环境：

作用域（栈内存 stack）
作用域的作用：给js提供运行环境，存储基本数据类型；

2.1-1 作用域的分类：

作用域（scope）：js代码执行的执行的环境，另外还可以存储基本数据类型的值。

  • 1.全局作用域：浏览器打开一个页面的时候，首先形成一个全局作用域，用来执行全局中的代码，全局作用域就是window对象（全局对象的一个实现，在浏览器中是window，在Node.js中是Global）
    • 全局作用域下定义的变量是全局变量，在全局作用域下都可以使用（前提是变量已完成定义），全局变量会给window加一个键值对（属性名是变量名，属性值是变量储存的值）
    • 不带var也可以给window加一个键值对，也是全局变量；在私有作用域中不带var添加的是全局变量
    • 全局变量window 下的属性对象可以直接使用（即window可以省略）
  • 2.私有作用域（函数作用域）：函数执行时形成的作用域;
    • 在私有作用域中定义的变量、形参、函数都是私有变量，在私有作用域外获取不到，但私有作用域可获取全局变量
  • 3.块级作用域：把代码块（if、for循环的代码块…）变成作用域，功能和私有作用域类似，是ES6新增；
    • 就是带{}的语句，但是需要注意的是对象（ var obj = {} ）中的 {} 是定义的格式，在这个 {} 中不能执行代码，不存在域的概念，不是块级作用域而是数据类型的一种格式

      私有变量和全局变量：

  • 全局变量：就是在全局作用域中声明的变量
  • 私有变量：在私有作用域（函数作用域）声明的变量以及函数形参

变量提升只发生在当前作用域中，函数不执行函数里面的私有变量不提升。

js的运行机制：

  • 1.js代码执行之前，浏览器会为其开辟全局作用域，然后对全局中的变量进行提升操作，对带var和带function进行提前的声明或者定义；
  • 2.变量提升结束后，js代码开始从上到下执行；执行的过程中对带var的变量进行赋值：
    • 1）如果是赋值一个基本数据类型的值，那么直接把基本类型的值存在作用域中，并且把变量和值关联起来；
    • 2）如果是赋值一个引用数据类型，浏览器会开辟一个堆内存，存储这个引用数类型的值，然后把这个堆内存的地址赋值给变量（这个时候这个内存地址是存在作用域中的）
  • 3.如果执行过程遇到函数执行会经历以下几步：
    • 1）浏览器会函数代码执行再开辟一个新的作用域（一块新的栈内存）→ 这解释函数每次执行都是互相独立的，因为每次执行的环境不同
    • 2）形参赋值 → 形参也是变量，把函数执行时实参赋值给形参。
    • 3）私有作用域中的变量提升，对私有作用域中的带var和带function的提前处理（在当前作用域中变量提升）
    • 4）从上到下执行函数代码
    • 5）作用域销毁 → 也有作用域不销毁的特殊情况
  • 4.如果再遇到函数执行重复第三步

2.2 变量重复声明

同名变量多次声明，只会声明一次，值是最后一个

变量提升时function的优先级高于普通变量

  • 1.变量多次声明时，浏览器不会多次重复声明，只会声明一次
  • 2.当函数名和变量重名时，函数的优先级比变量的优先级高。

2.3 变量提升的细节问题

  1. 判断条件中，var定义的变量不管条件成立与否都会进行变量提升；但function定义的函数：

    • 1.1 在ie10及以下版本浏览器中既提前声明又提前定义
    • 1.2 在ie10以上的浏览器中只提前声明不提前定义，只有判断条件成立后，函数体代码执行之后，函数才有值
    • 1.3 在新版浏览器中判断条件内写一个函数只当作判断使用，不会声明、定义；而老版浏览器中在条件语句中，声明函数，如果条为false，不会给函数变量赋值；如果为true才会给变量赋值；

  2. 变量提升只发生在等号的左边，等号右侧的不会进行变量提升，即使右侧是一个函数也不会提升；

  1. 变量名重复，不会重复声明，但会重复定义

  2. 函数return 右边的值不会进行变量提升，即使是函数也不行；（函数的返回值不会发生变量提升）

  1. 函数return 下面的代码虽然不执行，但是仍然会变量提升；

  1. 自执行函数不会进行变量提升（定义和执行在一起）

  2. let、const定义不进行变量提升且变量名不可以重复

  3. 函数与变量重名，函数优先级高于变量

2.4 带var和不带var的区别

  1. 在全局作用域中用var和function声明的变量，相当于为window上增加了一个同名属性；

  2. in 运算符：检测一个对象中是否有某个属性，如果有返回true，没有就返回false；

  3. 带var和不带var: 

    带var的是变量，会变量提升，不带var的不是变量。带var的在赋值之前使用不会报错，只不过值undefined；不带var不能在赋值之前使用，使用就会报错。

2.5 作用域链（Scope Chain）

变量的查找机制

作用域（scope）：js代码执行的执行的环境，另外还可以存储基本数据类型的值。

全局作用域：打开一个页面的时候，浏览器就会创建一个全局作用域（window对象）

私有作用域（函数作用域）：函数执行的时候浏览器开辟的，只要函数执行就会新开辟一个私有作用域；

块级作用域：把代码块（if/for循环…的代码块变成作用域），功能和私有作用域类似，是ES6新增的； 

作用域链：变量的查找机制；使用变量时，浏览器会在当前作用域中查找，如果找到就使用私有变量（形参、私有变量）。如果找不到，向上级作用域查找，如果找到就使用上级作用域中的，如果还没找到就继续向上查找，一直找到window为止；如果找到window还没有，此时后两种情况：
        如果取值（访问一个变量）：报错
        如果是赋值：给window添加一个属性

如何确定上级作用域？

看这个函数在哪个作用域定义的，定义时所在的作用域，就是该函数的上级作用域

3. 堆栈内存创建、释放以及闭包

JS引擎中内存分类（浏览器在执行代码时会形成两个虚拟的内存）
1. 堆内存：存储引用数据类型的值
2. 栈内存（作用域）：提供js代码执行的环境，存储基本数据类型的值

常见问题：

1. 函数执行时开辟的栈内存是在全局的栈内存开辟的，它会消耗全局作用域的空间；函数的作用域特别多的话会影响代码执行的效率
2. 堆内存特别大也会影响浏览器执行的效率

3.1 堆内存的创建和释放

创建堆内存：

创建一个引用数据类型的值（对象、数组、正则、日期、函数），浏览器会（默认）为这个引用数据类型创建一块堆内存，这个堆内存就是存这些引用数据类型的值；

释放堆内存：

将所有引用该堆内存的变量或者属性全部置为null（要等浏览器空闲的时候才会去垃圾回收，释放这些没有被引用的内存）；浏览器也会检测，当引用数据类型不被外界所占用时，将所占空间回收

一些浏览器的机制：

1.firefox和ie 计数机制：堆内存的每次占用会给计数加一；每次取消占用减一；计数为零时，堆内存回收
2.google 每隔一段时间检查一次，如果引用数据类型占的堆内存不被外界占用，堆内存回收

3.2 栈内存的创建和释放

栈内存（作用域）：供js代码执行的环境，存储基本数据类型的值；

创建栈内存：

1.浏览器打开一个页面时，会首先形成一个顶层的栈内存，就是全局作用域window
2.函数执行时产生的私有作用域，就是一个供函数代码执的栈内存

注意：仅仅创建一个函数是不会形成一个新的栈内存，函数执行的时候才会有栈内存

栈内存销毁：

1.顶层栈（全局作用域）销毁：页面关闭 全局作用域销毁；

2.普通函数：函数执行完后（没有返回值/返回值是基本数据类型的值/返回值是引用数据类型的值但不被外界接收和占用），立刻销毁

3.不销毁作用域：1)当函数执行完，如果函数返回了一个引用数据类型的值，并且被外面的变量或者属性接收，此时函数执行的作用域不能销毁，在这个作用域中储存的值也不会销毁；2)函数内部的引用数据类型值，被函数外的变量或者属性占用，此时函数作用域不能销毁；

4.不立即销毁作用域：函数执行的返回值是个函数，外层函数执行时得到里面的函数再执行，这样外层函数执行完后不会立即销毁，而是会等到里面的函数行完后再销毁

3.3 累加计数

需求：

累加计数；页面中有一个按钮，有一个初始值0，每点击一次这个按钮就要给这个值累加1；

思路：

点击的时候累加，所以这个按钮需要绑定一个点击事件
这个数字能够累加，说明数字一定是保存在了一个不销毁的地方；
  1. 利用全局作用域不销毁，保存这个数字的上一次的值
  2. 私有作用域保存上一次的值。（关键在于产生一个私有作用域，只有函数执行才会有私有作用域）
  3. 私有作用域不销毁（自执行函数）
  4. 块级作用域：块级作用域和私有作用域很类似，在块级作用域中声明的变量不能被外面访问；使用块级作用域时需要let、const等声明变量，这个代码块才会被解析成块级作用域。

3.5 闭包

专业解释：函数执行形成私有作用域，在私有作用域中声明变量不受外界影响，这种机制叫做闭包；

常见解释：形成一个不销毁的作用域，才是闭包；

闭包的作用

    1. 函数柯里化（柯利化函数）：把多参数的函数变成单参数的函数
    1. 利用闭包机制隔离全局命名空间：在过去没有模块规范的时候大家通常用这种方式区分命名空间
    1. 惰性封装：
    1. 利用闭包不销毁作用域保存数据：累加计数、选项卡的闭包解决方案

4. this

this是JS的关键字，代表当前代码执行的环境对象。
一般在函数中使用，并且是在函数执行时，根据函数的不同执行方式确定不同的值。

this在运行时不可以赋值

this的几种情况：

    1. 全局作用域下，this指向window；this和window的空间地址是一样的；window下如果没有某一个属性，undefined
    1. 事件函数中的this指向绑定该事件的元素；（this等于触发事件的函数）
    1. 方法调用（函数执行）时，看方法前面是否有点 . 如果有，点前面是谁，方法中的this就指向谁，如果没有，方法中的this就指向window；如果有多个点，也只看它执行前面的那一个就可以了
    1. 自执行函数中的this指向window
    1. setTimeout/setInterval() 定时器回调函数中的this指向window
    1. 构造函数中的this指向实例（原型上的方法中的this不一定指向实例，还需要看这个方法是谁调用的）
    1. call、apply、bind可以改变this指向 1）call改变this指向，call中的第一个参数赋值给函数中的this，剩下的参数一次赋值给形参
      2）apply改变this的指向，apply只有两个参数，第一个参数赋值给this，第二个参数是一个数组，数组中的每一项当成参数赋值给函数fn
      3）bind是预处理this，用法和call一样，但是不会执行函数，bind的执行结果是一个新函数，新函数执行才会改变this指向
    1. 箭头函数中的this指向函数定义时所在作用域（上级作用域）中的this

箭头函数

ES6新增的语法：省略function关键字，在形参入口后增加 => 箭头，后面紧跟函数体；

箭头函数的简化语法

  1. 只有一个形参时，可以省略 形参入口的小括号
  2. 如果函数只有一行代码，或者只有return 指定返回值，可以省略函数体的花括号和return关键字
