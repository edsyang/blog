1. DOM2级事件（低版本IE不兼容）

1.1 DOM0级事件：（DOM1级事件）

元素对象.onXxx = function () {}

通过给元素对象的事件属性（onXXX）赋值一个函数实现这个事件的监听；当事件触发的时候，浏览器就会把这个事件监听函数执行

DOM0级事件不能给同一个事件绑定多个监听函数，因为DOM0级事件是给对象的事件属性赋值，一个属性只能保存一个值，多次赋值，属性代表的是最后一个值

DOM0级事件解绑事件监听函数：把对象的事件属性赋值为null

DOM0级事件都是绑定在冒泡阶段的

1.2 DOM2级事件：

DOM2级事件绑定：元素对象.addEventListener('不带on的事件名', 事件函数, 是否捕获阶段)
不带on的事件名：click、mouseenter、mouseleave...
事件函数：事件触发时执行的函数
是否为捕获阶段：true表示绑定在捕获阶段，false表示绑定在冒泡阶段；默认值为false

DOM2级事件解绑事件监听函数：元素对象.removeEventListener('不带on的事件名', 事件函数, 是否捕获阶段)
事件名：不带on的事件名
事件函数：绑定的哪个函数，移除的时候就要移除哪个函数【一定是同一个函数】
是否捕获阶段：true捕获阶段，false冒泡阶段

DOM2级事件相对于DOM0级事件的好处：
1. DOM2级事件可以给同一个元素的同一个事件绑定多个事件监听函数（但是不能重复绑定同一个监听函数），并且在事件触发的时候，会按照绑定顺序依次触发
2. DOM2级事件可以选择绑定在捕获或者冒泡阶段
3. 绑定和解绑DOM2级事件都有专门的方法

标准浏览器的DOM2级事件可以绑定多个事件监听函数的原理：
DOM2级事件给每个元素的每个事件都设置了一个事件池，类似于一个数组。每次addEventListener都是把事件监听函数加入到事件池中（把事件监听函数放到事件池中并不会立即执行），等事件触发时，再从事件池中把事件监听函数按顺序拿出来一个一个执行

IE DOM2级事件处理方法：
绑定DOM2级事件：元素对象.attachEvent(带on的事件名, 事件函数)
解绑DOM2级事件：元素对象.detachEvent(带on的事件名, 事件函数)

IE DOM2级事件和普通DOM2级事件区别：
1. 绑定和解绑的方法不一样
2. IE和DOM2只能绑定在冒泡阶段
3. IE在给同一个元素同一个事件绑定多个监听函数时，不一定按照绑定顺序触发

function () { console.log(1) } 和 function () { console.log(1) } 虽然长得相同，但是这两个函数并不是同一个函数！！！因为两者的空间地址不相同。操作一个对另一个没有任何影响，只有当操作这个函数的空间名的时候才是操作同一个函数

1.3 DOM0和DOM2事件的区别

  1. DOM0事件绑定是给元素的事件属性赋值，一个属性只能对应一个值，所以不能给同一个事件绑定多个监听函数；DOM2有事件池，所以可以绑定多个
  2. DOM2可以选择绑定在冒泡阶段还是捕获阶段；DOM0只能绑定在冒泡阶段

2.1 原生JS模拟实现拖拽

项目中的事情、岗位职责：
1. 产品经理（项目经理、运营）PRD - 产品需求（需求就是一个功能或者一个效果）文档
2. 前端+后端 工程开发（也叫技术）
3. QA（测试）测试做出来的东西是不是符合需求，是不是能用；这帮人会提bug

需求：页面中有一个盒子可以拖拽到任意地方（盒子跟着鼠标动）

  • 1.在盒子上点击鼠标左键不松开（onmousedown）
  • 2.按住鼠标左键不松开，移动鼠标（onmousemove）时，盒子要跟着鼠标移动（鼠标跟随）
  • 3.等到达目的地后，松开鼠标（onmouseup），盒子要停在松开鼠标的地方；再移动鼠标，盒子不能再跟随鼠标

思路：根据上面的功能描述，整个拖拽分为三个阶段

  • 1.鼠标按下去的时候给盒子赋予可以跟随鼠标移动的能力 - 需要监听盒子的onmousedown事件，可以跟着鼠标移动是给元素绑定了onmousemove事件

  • 2.按住左键不放，移动鼠标会触发元素的onmousemove事件，在事件触发时，要计算盒子的位置，并且把位置设置给盒子，实现盒子跟随鼠标

  • 3.松开鼠标后，盒子要停下来，不会再跟随鼠标移动，说明盒子跟随鼠标移动的能力在鼠标抬起时被取消，所以把onmousemove事件移除即可

    拖拽鼠标丢失问题原因：

    拖拽并不是鼠标真正的拖着盒子走，而是鼠标走鼠标的，盒子不断的跟随鼠标；

    在鼠标移动的过程中，不断计算盒子应该出现的位置，这个计算是需要时间的，如果在计算的一段时间内，再次快速移动了鼠标，因为上一次还没计算完，盒子还没到达上一次的位置，此时移动鼠标就会出现盒子跟不上鼠标了，元素没跟上鼠标，再移动鼠标就不会触发元素的onmousemove事件，也就不会再跟随鼠标了；

    当鼠标丢失元素后，再松开鼠标时，触发的就不是元素的onmouseup事件了，而我们在元素的onmouseup事件函数中移除元素的onmouseup的操作没执行，导致元素的onmousemove一直存在，这就导致了鼠标再次回到元素上面时，盒子还能跟着鼠标移动

解决方案：

  1. 把元素和鼠标绑定在一起 元素对象.setCapture() 方法；当拖拽结束后在解绑 元素对象.releaseCapture() 方法；【Chorme不兼容，Firefox、IE可以使用】
  2. 事件委托解决问题：因为元素很小，只有鼠标在元素上面移动的时候才会触发这个元素的onmousemove事件，只要动的快，鼠标和元素分开后就不会再触发元素的onmousemove事件，所以把onmousemove和onmouseup事件绑定给document；

    注意：document.onmousemove = dragMove直接把事件绑定给document，导致了dragMove函数中的this变成了document（事件函数中的this指向绑定当前时间的元素），但需要的是让dragMove中的this指向被拖拽的元素，所以需要修改dragMove中this的指向：document.onmousemove = dragMove.bind(this) ；bind方法是返回了一个修改this后的新函数【不会执行】

2.2 原生拖拽（拖放）

1. HTML5原生支持拖拽，但是需要给被拖动的元素设置行内属性 `draggable = "true"`
2. 被拖拽元素和放置元素
    2.1 被拖拽元素：被拖拽的元素（draggable = "true"）
        1) 被拖拽元素.ondragstart = function(){} 拖拽开始时触发
        2) 被拖拽元素.ondrag = function(){} 在拖拽时触发
        3) 被拖拽元素.ondragend = function(){} 拖拽结束时触发
    2.2 放置元素：将来被拖拽元素放置的元素
        1) 放置元素.ondragenter = function(){} 当被拖拽的元素进入放置元素时触发 
        2) 放置元素.ondragover = function(){} 当被拖拽元素经过放置元素时，触发放置元素的该事件（如果要使用放置元素的ondrop事件，必须阻止ondragover的默认行为）
        3) 放置元素.ondragleave = function(){} 当被拖拽元素离开放置元素时，触发放置元素的该事件
        4) 放置元素.ondrop = function(){} 当被拖拽元素在放置元素上松开鼠标（结束拖拽）时触发；当向放置元素拖入文件时，阻止浏览器打开文件，需要在ondrop中阻止默认行为

原生的拖拽是基于原生拖拽事件完成的

3. 发布订阅

3.1 发布订阅准备

  • DOM2级事件给每个事件、每个元素都设置了一个事件池（类似于数组的东西），每次addEventListener()都是向事件池中添加对象，等到DOM2事件真正触发时，把函数从事件池中一个一个拿出来执行一下；removeEventListener()是取消订阅，把某个已订阅的事件函数从事件池中移除掉；所以可以实现给一个元素的同一个事件绑定多个事件函数
  • JS中的事件是元素或者浏览器发生特定的交互瞬间；也就是说事件就是一个时刻
  • 事件思想：JS中所有可以量化的时刻都可以称为一个事件；例如5s后、动画结束后都是一个具体的时刻，所以5s后和动画结束后都可以叫做事件

    思考：既然5s后和动画结束后都是事件，那么怎么监听这些事件？事件发生时怎么让事件函数执行？

3.2 发布订阅

发布订阅：模拟DOM2级事件的事件池思想，当某一个时刻到来需要做很多事情（执行很多函数），解决自定义事件的；准备一个数组当作事件池，并提供向事件池中添加（添加订阅）以及删除（取消订阅）函数的方法，并且还要提供一个执行事件池中的函数的方法（DOM2级事件是内置的事件，事件发生时浏览器自动执行事件函数，但是自定义的事件需要手动执行事件函数）等到该时刻到来时把事件池中的函数一个一个执行了；这种解决问题的方法称为发布订阅模式（观察者模式）

  • 订阅：订阅某时刻到来时，把想做的事情放到事件池中
  • 发布：等到时刻真的到来，把事件池中的方法都执行了
