1. js的引入方式

无论是内嵌式还是外链式，都建议把script标签写在body结束标签的前面。

  • 行内式
    通过向html元素行内添加一些js代码，如给div绑定点击事件
  • 内嵌式
    在html文档中写一个script标签，把js代码写在script标签内
  • 外链式
    在html文档中写一个script标签，并且设置src属性，src的属性值引用一个js文件资源，而js代码写在js文件中；

    使用外链式引入js文件时，script标签里面不能再写js代码了，即使写了也不会执行。

2. js的输出方式

  • alert方法(函数):

    在浏览器中通过弹窗的方式输出（浏览器提示框）
浏览器弹出警告弹框，他会阻塞代码执行；
alert弹框，显示的内容，会默认转换成字符串显示；

  • console.log方法(函数)：

    在浏览器控制台输出日志。console是浏览器的开发者工具的一个页卡（在浏览器中按下F12按键【部分电脑需要按下Fn和F12】）
在控制台打印输出；附带文件信息和行号
可以打印多个数据，数据之间用逗号隔开

  • innerHTML/innerText属性：

    修改元素中的HTML或者文本；

  • prompt()方法(函数)：

    弹出一个可输入的弹框，还有两个按钮（确认和取消）；
点击确定，返回你在输入框输入的内容；
点击取消，返回空(null)；

  • confirm()方法(函数)：

    确认弹框，有两个按钮，确认，取消；
点击确认，返回true
点击取消，返回false

  • document.write

  • console.dir：打印详细信息

3. js的组成

- ECMAScript（ES）：js的核心语法

- DOM：Document Object Model文档对象模型，提供各种API（属性和方法）让JS可以获取或者操作页面中的HTML元素（DOM元素） 它提供了，js改变html的一些属性和方法

- BOM：Browser Object Model 浏览器对象模型，提供各种API让js可以操作浏览器  它提供了，js操作浏览器的一些属性和方法

4. 变量（variable）

它不是具体的值，只是一个用来代表某个值的标识符，因为它表示的值可以改变，所以称为变量。

基于ES语法规范，在JS中创建变量有以下方式：
- var（ES3) 创建普通变量
- function （ES3）创建函数（函数名也是变量，只不过存储的值是函数类型而已）
- let （ES6）声明变量
- const （ES6）创建常量（常量就是不变的值，如光速就是常量）
- import （ES6）基于ES6的模块规范导出需要的信息
- class （ES6） 基于ES6创建 类

4.1 命名规范

创建变量，命名的时候要遵循一些规范：

- 严格区分大小写
- 遵循驼峰命名法
- 按照数字、字母、下划线或者$来命名（数字不能作为开头），
- 命名的时候基于英文单词拼接成一个名字（第一个单词字母小写，其余每一个有意义单词的首字母都大写）
- 不能使用关键字和保留字：在js中有特殊含义的叫做关键字，未来都可能成为关键字的叫做保留字

5. 数据类型

数据类型是一门语言进行生产的材料，JS中包含的值有以下这些类型：

- 基本类型：
    - 数字 Number
    - 字符串 String
    - 布尔 Boolean
    - Null
    - Undefined
    - Symbol 表示一个唯一值（ES6新增）
- 引用数据类型：
    - 对象 Object
        - 普通对象 Object
        - 数组对象 Array
        - 正则对象 RegExp
        - 日期对象 Date
        - 数学对象 Math
        - ...
    - 函数 Function

基本数据类型和引用数据类型的区别:

基本数据类型保存在栈内存中，保存的信息少一点
引用数据类型保存到堆内存中，保存的信息多一点

为什么会有这种情况？

基本数据类型（也叫做值类型）的操作，直接操作就是这个值，意思就是说变量本身就代表这个值。所以： var b = a; 存储12这个值，然后声明一个变量b，然后让变量b和这个新的12关联起来。这一行和 var b = 12本质上没有任何区别；

引用数据类型都是存放在堆内存空间中的，同时这个堆内存空间有一个十六进制的内存地址。我们在声明一个引用数据类型时，不是直接把对象赋值给变量，而是把对象的堆内存地址赋值给变量。

所以引用数据类型的操作不是直接操作的值，而是操作它的引用数据类型的堆内存地址。所以var obj2 = obj1; 只是把obj1代表的堆内存地址地址赋值给了变量obj2。

所以obj2.age = 100;是通过obj2的堆内存地址找到堆内存地址中的存储的age的值，把它修改成100。同时，我们访问obj1.name 时也是先通过obj1存储的堆内存地址找到内存空间，然后从里面把属性age的值取到，此时这个内存空间中的值已经修改成100了。所以obj.name 也是100

6. 数据类型详解

6.1 Number

数字类型；包含数学数字，正数、负数、0、NaN(not a number)：可以进行数学运算

Number()

Number(需要转换的值); 是强制转换
将其他数据类型转换成数字类型；
转换过程中如果是有效数字，得到的是一个数字；
引用数据类型转数字，先调用引用数据类型的toString()方法将其转为字符串，再用Number转为数字

有效数字检测

isNaN(需要检测的值)：检测当前是否是一个有效数字
语法：isNaN(需要检测的值)
如果是一个有效数字(不是NaN)，isNaN执行后就会得到一个false
如果不是一个有效数字(是NaN)，isNaN执行后就会得到一个true
(我们得到的这个false或者true称为isNaN函数的返回值【就“结果”的意思】)

数据类型转换

parseInt、parseFloat方法：

作用和Number方法相似，都是把非数字类型值转换为数字，区别在于Number方法是强制转换，即要求被转字符串中必须都是数字，如果不满足这一条件直接得到NaN，parseInt和parseFloat则是非强制转换。
parseInt：从字符串中截取整数；从左到右解析字符串，遇到非有效数字时停止检测，将得到的数字返回，如果第一位不是一个有效数字，返回NaN
parseFloat：从字符串中截取含带小数部分的有效数字；它比parseInt多识别一个小数点；
bonus：toFixed()：保留小数;返回结果是一个字符串语法：(数字).toFixed(保留小数点后几位);

数据类型检测

typeof
语法：`typeof 被检测的值` 或 typeof()   返回被检测值的数据类型；

typeof 是有局限性的，检测基本数据类型时，typeof检测null会返回object，因为null是空对象指针。同时，typeof检测引用数据类型时，不能具体区分是哪一种对象数据类型（对象、数组、正则等）只会统一返回object。

6.2 Boolean

布尔类型值只有两个值，true和false；布尔运算用来测试真假值，即运算结果是真的还是假的，通常结合js的判断语句（if/switch-case/三元表达式）使用。

Boolean();方法

语法：Boolean(需要转换的值) ；得到转换后的值

!运算符（取反运算符）

语法：!需要取反的值 ；
得到取反后的布尔值。其内部机制是先把其他数据类型转换成布尔值，然后再取反。
true取反就是false，而false取反就是true

运算符是有固定功能的关键字或者符号。它通过操作值也可以得到返回结果，与方法不同的是运算符不需要小括号。

转换规律：在js中只有 0/NaN/空字符串’’/null/undefined 这5个值转换成布尔值是false，其余的都是true。

!! 运算符 等效于Boolean方法(隐式的将其他数据类型转换成Boolean值)

语法：!!需要转换的值；

逻辑运算符 || 和 &&

- && 表示并且，两个值中必须两个都是true，才返回true
返回值：如果第一个为true返回第二个，如果第一个为false返回值第一个(遇false返回)
- || 表示或者，两者中有一个为true，就返回true
返回值：如果第一个为true就返回第一个，如果第一个不为true就返回第二个(遇true返回)

6.3 String

在js中用单引号''或双引号""包裹的文本都是字符串

String();将其他数据类型转成字符串；

- 基本数据类型转成字符串，直接在外层包裹""
- 引用数据类型
    - 对象：对象转字符串，不论对象有没有内容，都是"[object Object]"
    - 数组：将数组的中括号去掉，然后包裹引号

+ 拼接符

将字符串拼接；其他数据类型和字符串拼接时，会将其他数据类型先转成字符串然后拼接

- * / %：将两边转换成数字然后进行运算；

索引

给字符串进行排序，从左边第一位开始依次累加，第一位是0
length：表示字符串的长度，字符串有几个字符;
length-1:等于字符串最后一位索引的值

6.4 Null和Undefined

Null和Undefined区别：

Null：现在没有以后可能会有，占位；null类型只有一个值就是null；
Undefined：现在没有以后也不会有；undefined类型也只有一个值就是undefined；

Null

空对象指针；不占内存，通俗理解就是人为的手动先赋值为Null，后面程序中我们会再给它赋值为其他值；

常见几种Null的情况：
    1.通过id获取元素时，如果没有获取到元素，返回Null；
    2.正则捕获的时候，如果没有捕获到内容，返回到Null；
    3.想要释放引用数据类型的占用时，将变量的值赋值成Null；
    ....

Undefined

未定义。多数情况是某些浏览器内置机制设置的默认值，声明一个变量不赋值，这个变量的默认值就是Undefined

Undefined常见的几种情况；
    1.变量只声明，未赋值，变量中的只是Undefined；
    2.函数中有形参没有对应的实参时，形参默认赋值Undefined；
    3.函数中，如果没有返回值，函数的执行结果就是Undefined；
    4.获取对象下的某个属性时，属性不存在，获取的结果就是Undefined；
    ....

6.5 Object(对象)

6.5.1 普通对象

无序的键值对集合

  • 由花括号包裹起来的{key : value}
  • 由零到多组属性名和属性值（键值对）组成。属性名可以是数字、字母、或者下划线等，并且属性名都是字符串类型的。

属性是用来描述当前对象特征的，属性名是当前对象具备的特征，属性值是这个特征的描述，一组属性名和属性值成为一组键值对

声明对象

定义对象的步骤：
1.在堆内存中开辟一块空间，浏览器会给空间分配一个16进制的地址；
2.将对象的键值对保存在空间中；
3.将空间地址赋值给变量；

对象中的属性名都是字符串类型的

对象的操作：键值对的增删改查

  • 获取对象的某个属性的值
    • 对象.属性名
    • 对象[‘属性名’]

方括号里面还可以写变量或者表达式

如果用. 后面的属性名不用写引号，如果使用[]，属性名需要用字符串包裹

如果属性名是数字，只能使用 []

如果获取一个对象不存在的属性，会得到一个undefined

  • 增加/修改

js对象中的属性名是不允许重复的，是唯一的

如果给一个对象的属性赋值，有两种情况:
- 如果对象之前不存在这个属性，那么就是给这个对象增加一个属性，值是等号右侧的值
- 如果对象之前已经存在这个属性了，再赋值就是修改对象中的这个属性的值
  • 删除
    • 彻底删除：把对象的属性名和属性值都删除。 delete obj[‘name’]
    • 软删除：将对象的某个属性的值赋值为null。 obj.name = null

对象属性名的命名规则

  • 通过”.”添加的属性，属性名的命名规则和变量是一样的；
  • 通过”[ ]”添加的属性，属性名是字符串就行；

6.5.2 数组对象(Array)

有序的键值对集合

数组是【有序】的键值对集合；但是数组的键是浏览器设置的，不需要我们手动设置，并且键都是从0开始的数字，我们称数字属性名为【索引】。数组的第一项的对应的索引是0，第二项对应的索引是1，以此类推，第n项对应的索引是 n - 1。

数组的声明

数组的定义步骤：
1.首先在堆内存，开辟一块空间；
2.将数组的成员项，键是索引，值是数组的成员项，保存在空间中；
3.把空间地址赋值给变量；

索引

给数组的成员项排序编号，从0开始；从左向右依次递增；

length：数组长度，表示数组成员个数；

length-1：最后一个数组成员项

数组的操作

数组的键是数字，所以只能用方括号的方式获取、修改，而且写在方括号里面的属性名不需要用引号包裹；

7. 检测数据类型

typeof 操作符

用法：typeof 要检测的数据 || typeof(要检测的数据)

检测结果：返回的是一个字符串，字符串的内容是被检测数据的类型

局限性：检测基本数据类型时，typeof 检测 null 会返回 Object，因为 null 是空对象指针。同时，typeof 检测引用数类型时，不能具体区分是哪一种对象数据类型(对象、数组、正则等)只会统一返回 Object

instanceof 操作符

用法：要检测的数据 instanceof 数据类型

检测结果：返回的是一个布尔值，检测当前对象是否属于某个类

局限性：只能检测引用数据类型，且只要原型链上出现过构造函数都返回 true，检测结果不太准确(无法基于这个结果判断是否为普通对象)

constructor 属性

存在于构造函数的原型上，指向构造函数，对象可以通过 __proto__ 在其所属类的原型上找到该属性

用法：要检测的数据.constructor === 数据类型

检测结果：返回的是一个布尔值

局限性：比较准确的判断对象的所属属性，但如果有继承的话，检测结果不太准确

Object.prototype.toString.call() 方法

检测数据类型中最为准确的

在 Object（基类）的原型上定义的这个 toString() 方法

用法：Object.prototype.toString.call(要检测的数据)

检测结果：返回的是一个字符串 [object 被检测的数据类型]

局限性：只能检测内置类, 不能检测自定义类, 只要是自定义类都会返回 [object Object]

原理

  1. 每一种数据类型的构造函数的原型上都有 toString 方法
  2. 除了 Object.prototype 上的 toString 是用来返回当前实例所属类的信息(检测数据类型的), 其余的都是转换为字符串的
  3. 对象实例.toString(): toString 方法中的 this 是对象实例, 也就是检测它的数据类型, 也就是 this 是谁, 就是检测谁的数据类型
  4. Object.prototype.toString.call([value]) 所以是把 toString 执行, 基于 call 改变 this 为要检测的数据值

8. 数据类型的比较

=

赋值号：把等号右边的值赋值给左边

==

相对比较：等号两边的数据比较，返回的是布尔值(相等返回true，不相等返回false)
如果变量的数据不一样，先转换成数据类型再比较；

== 中不同类型之间比较的规律：

类型 类型 规律 其他
对象 对象 比较空间地址
对象 数字 对象→字符串→数字后再比较
对象 字符串 对象→字符串后再比较
对象 布尔 对象→字符串→数字；布尔→数字；然后比较 true→1；false→0
数字 布尔 布尔→数字后再比较 true→1；false→0
数字 字符串 字符串→数字(通过Number())后再比较 空字符串是0
布尔 字符串 布尔→数字；字符串→数字；然后比较
null undefined 返回true
null、undefined 其他类型 都是false 没有toString()方法
=== 
绝对比较：先比较类型，再比较值

!= 

不等号：和"=="得到的结果正好相反

!==

绝对不等(不全等)：和"==="的取值正好相反
