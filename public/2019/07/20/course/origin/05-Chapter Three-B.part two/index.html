5. 面向对象（OOP：Object Oriented Programming）

5.1 面向过程

以过程为核心，研究现在要解决的问题，既不考虑以前，也不考虑将来，这段代码就解决现在的问题。如果以后再有相同的功能，就再写一遍相同的代码

5.2 面向对象

是一种对现实世界的理解和抽象的方法。面向对象关心现在的功能分类解决，现在解决过的问题，我们过往有没有类似的代码可以复用，我们现在的写的代码能不能给将来用。（对现在的问题分类解决，给未来的问题提供借鉴）

5.3 面向对象的研究范畴

1. 对象：万事万物都是对象，每个对象都具备各自的属性和功能
2. 类：抽象事物的特性和特点，把对象分成不同的类型，例如动物类、植物类；类是一个是描述一群事物共同特点的抽象概念
3. 实例：类中的一个具体的个体。这个个体既然属于类，那么这个个体一定具有这个类的所有的特性和功能

例如：人类就是一个类，人类最显著的特点是可以制造并使用工具。人类的属性有语言、智慧、吃饭、睡觉.... 每个人都是人类的一个实例，每个人都有这个类型中的所有的特性和属性。

5.4 面向对象的体现

数据类型，其中强调的就是类型。例如数组类、普通对象类，这是因为数组类和对象的属性和方法不同。

例如数组是有序的键值对集合，还可以push，pop、splice等，而对象是无序的键值对集合，而且对象不可以pop

而 var ary = [1, 2] 是数组的一个实例，所以ary有所有数组的特性

而 var obj = {name: 1} 是对象的一个实例；

5.5 JS中的内置类

  • Object 对象类（基类）
  • Array 数组类
  • Date 日期类
  • RegExp 正则类
  • Function 函数类

这些内置类都是函数数据类型的

5.6 面向对象的研究范畴

研究对象、类、实例三者之间的关系

讨论封装、类继承和多态（重载、重写）

1.封装：
    由面向过程向面向对象推进；将同种对象的属性、方法抽象成一个类，然后通过类的属性、方法访问类
    目的：为了创建对象
2.继承：
    在封装的基础上，将同类事物继续抽象，继承时，子类拥有父类的属性、方法；并且可以有自己特有的属性、方法
3.多态（重载、重写）：—— js没有真正的重载
    不同种对象之间行为的抽象

5.7 ！面向对象的总结！

1.类：类都是一个函数数据类型（Function），每个类都有一个原型（prototype）
2.原型（prototype）：每个类的原型都是对象，并且有一个特殊的属性constructor，它的值指向当前类的构造函数本身
3.对象：（实例对象、原型对象、函数对象...）都有一个特殊的属性__proto__，它的值指向当前对象（实例对象、原型对象、函数对象...）所属类的原型（prototype）

创建一个函数，将私有属性写在函数里面(this.xxx = xxx)，公有属性放到prototype上(xx.prototype.XXX = 方法、属性)，再new出来这个构造函数即可

6. 面向对象-封装

由面向过程向面向对象推进；将同种对象的属性、方法抽象成一个类，然后通过类的属性、方法访问类

私有属性：添加到this上的属性（构造函数解决）
公有属性：将实例公有属性的属性、方法放到一个公共的位置（原型模式解决）

6.1 单例模式实现模块化开发、高级单例模式及工厂模式

6.1.1 单例模式

把描述一个对象的属性放到一个对象中，这种封装方式称为单例模式。

对象对应的变量名称为命名空间（namespace）

优点：解决了全局变量互相覆盖的问题

6.1.2 高级单例模式

不是直接将一个对象赋值给变量名，而是在等号右侧写一个自执行函数，在自执行函数末尾返回一个对象

优势：1.可以选择哪些给外面用，哪些不给外面用；2.代码很简洁。

单例模式、高级单例模式解决了全局变量命名冲突、互相覆盖的问题。但是效率很低，不能批量的创建对象；当大规模创建对象时，就需要写许多重复的代码

6.1.3 工厂模式

把创建对象的细节封装成一个函数，在函数中为这个对象添加属性，这种创建对象的模式叫做工厂模式；

优势：可以批量生产

缺点：生产出来的对象都一样，没有类的概念

6.2 构造函数模式

6.2.1 构造函数模式

把创建对象的细节封装到构造函数中。通过this.xxx = xxx向实例对象上添加属性这种方式创建对象称为构造函数模式此时构造函数就叫做类，函数名叫类名(一般情况下类名首字母大写，如果有多个单词，使用驼峰命名法)

构造函数也是一个函数，但是其内部实现和执行（调用）方式和普通函数不同

1. 构造函数只能用new来执行（用new执行，这个函数才叫构造函数），命名要用大驼峰命名法（CamelCase），当前的函数名就是类名 ；声明函数时和普通函数一样
2. 构造函数不需要显式的手动创建实例对象，在通过new调用构造函数时，构造函数会隐式的创建一个属于这个类的实例对象，并且把构造函数中的this指向这个实例对象。后续的加工都在this上加工就可以了。
3. 构造函数不需要显式的手动返回实例对象，构造函数在执行后会隐式的返回一个当前类的实例对象；实例对象内的属性成为【私有属性】

不足：

不能把属于这个类共有的属性和方法共有。this.xxx = xxx 的方式把属性都添加到了实例上，而每个实例都是一个单独对象，这些对象之间无法共享属性和方法；

6.2.2 new 构造函数执行

构造函数通过可以通过new实例化一个实例

函数只要被new调用时就是构造函数。new 谁最后得到的就是谁的实例

new 构造函数执行机制：

1. 开辟作用域
2. 形参赋值
3. 变量提升
4. 隐式创建一个属于当前这个类的实例对象，然后把this指向这个实例对象。
5. 执行构造函数中的代码；如果this.xxx = xxx; 就是给实例添加私有属性。
6. 隐式返回这个实例对象，相当于 return this;
7. 销毁栈内存（构造函数的作用域销毁与否和普通函数一样）

普通函数执行机制：

1. 开辟私有作用域
2. 形参赋值
3. 变量提升
4. 代码执行
5. 销毁栈内存（特殊情况除外）

构造函数和普通函数的不同点

1. 构造函数中的this指向当前实例，而普通函数中的this的指向和函数执行有关
2. 构造函数中的函数名用大驼峰命名法-CamelCase（约定俗成，用来区分普通函数）
3. 构造函数执行时，有参数必须加小括号执行，没有参数（实参）小括号可以省略
4. 构造函数中return返回的实例不受基本数据类型的值的影响，但会打断函数的执行；但引用数据类型的值会把默认返回的this的指向改变

6.2.3 实例对象的创建方式

  • 字面量的方式（字面的意思）
    • 创建引用数据类型：{}就是对象，[]就是数组，/\d/ 就是正则；用字面量创建的引用数据类型和实例的方式创建的数据类型没有区别
    • 创建基本数据类型
  • 通过new 类型构造函数
    • 创建引用数据类型：通过实例的方式创建引用数据类型和字面量创建的没有区别；
    • 基本数据类型：使用实例方式创建基本数据类型，会得到一个对象；而对象不是基本数据类型的值；所以基本数据类型只能用字面量的方式创建；
  • Object.create

  • 通过对象直接量的对象使用Object.prototype作为原型
  • 通过new创建的对象使用构造函数的prototype属性作为原型

6.2.5 构造函数显式设置return

不要随意在构造函数中指定返回值

  • return 基本数据类型: 构造函数不受影响，仍然正常返回实例。
  • return 引用数据类型：构造函数隐式返回的实例就会被return的这个引用数据类型的值所覆盖，并且返回的值也不再是该类型的实例了。

6.3 基于构造函数的原型模式

原型的五大原则：

1. 所有的引用类型（数组、对象、函数）都具有对象特性，及可自由拓展属性（null除外）
    1）在原型对象中新添加（通过prototype添加属性的方式拓展原型）
    2）将原型的空间地址覆盖
        Ⅰ 自定义类可用新的空间地址将原来的prototype覆盖，但会导致自带的constructor丢失（可手动添加）
        Ⅱ 内置类的prototype的空间地址不能被覆盖，但可以新增
2. 所有的引用类型（数组、对象、函数）都有一个__proto__属性（隐式原型），属性是一个普通对象
3. 所有的函数都有一个prototype属性（显式原型），属性也是一个普通对象
4. 所有的引用类型（数组、对象、函数）__proto__属性值指向它的构造函数的prototype的值
5. 当试图得到一个对象的某个属性时，若这个对象没有这个属性，那么会去它的__proto__（即它的构造函数的prototype）中寻找 ———— 原型链

6.3.1 基于构造函数的原型模式

把一个类公有的属性、方法放在prototype（原型）对象中。这样做可以实现只要是这个类实例，就能够使用这些方法或者属性；这样做解决了公有属性和方法公有的问题。

prototype（原型）对象 - 显式原型

每个函数（普通函数、构造函数【类】）都天生自带一个属性prototype（原型），它的值是一个`对象`，用来存储这个类共有的属性和方法。存储在prototype上的属性和方法称为公有属性和方法；

constructor构造函数

prototype对象上天生自带一个constructor属性，它的值就是构造函数本身；或者说 prototype对象的constructor属性指向当前所属的构造函数；

__proto__属性 - 隐式原型

通过实例的__proto__，可以修改原型，最好不要修改，容易引发错误；可以拓展原型

每个对象都自带一个__proto__属性，是js内部使用寻找原型链的属性，__proto__属性指向当前对象所属类的原型

6.3.2 原型链

对象的属性查找机制

机制

当我们访问（对象.属性名）一个实例对象的属性时，浏览器会首先在实例对象的私有属性中查找，如果找到就使用这个私有属性，如果没找到，通过实例对象__proto__属性去当前实例对象所属类的prototype（原型）上查找，如果在原型的公有属性中找到这个属性，就使用这个公有的属性。如果找不到，就通过prototype（原型）对象的__proto__继续向上查找，一直找到基类Object的原型上，如果还没找到，就返回undefined

6.3.3.3 面向过程、面向对象比较

面向过程：

优点：性能比面向对象高，因为类调用时需要实例化，开销比较大，比较消耗资源;比如单片机、嵌入式开发、Linux/Unix等一般采用面向过程开发，性能是最重要的因素。 
缺点：没有面向对象易维护、易复用、易扩展

面向对象：

优点：易维护、易复用、易扩展，由于面向对象有封装、继承、多态性的特性，可以设计出低耦合的系统，使系统更加灵活、更加易于维护 
缺点：性能比面向过程低

面向对象显著的特性

1.程序设计的重点在于数据而不是过程； 
2.程序被划分为所谓的对象； 
3.数据结构为表现对象的特性而设计； 
4.函数作为对某个对象数据的操作，与数据结构紧密的结合在一 
5.数据被隐藏起来，不能为外部函数访问； 
6.对象之间可以通过函数沟通； 
7.新的数据和函数可以在需要的时候轻而易举的添加进来； 
8.在程序设计过程中遵循由下至上（bottom-up）的设计方法。

6.3.4 自定义类和内置类

  • 自定义类：写一个构造函数，把自定义类的公有属性和方法写在构造函数的原型上；
  • 内置类：Array / Date / RegExp / Function / Object

    所有的内置类都是函数数据类型

6.3.5 Function类：函数类

所有的函数（类、普通函数）都是Function类的实例

Array Number String Boolean Object Date Function 等...都是函数数据类型的值；所以这些内置类都是Function的实例

数据类型检测：instanceof

Object 和 Function的关系：

  • Function 的 prototype（原型对象）的 constructor 是构造函数本身
  • Function 是一个类，也是一个函数，所以 Function 是自己的实例
  • Object 是基类，Function 也是 Object 的实例；
  • Object（实例）的 constructor（构造函数属性）指向构造函数
  • Object 是 Function 的实例，实例的 __proto__（隐式原型）是原型对象
  • Function 是一个类，但 Function.prototype 是一个对象，它的 _proto_ 属性值指向 Object.prototype
  • Object 的 prototype 的 _proto_ 为 null

7. 面向对象-继承

子类继承父类，让子类的实例能够访问父类的属性和方法，把父类的私有属性或公有属性变成子类的私有属性或者公有属性；如让B类继承A类，B类为子类，A类为父类（超类）

Question：如何能让b能够调用到A类上say这个方法？

能让b调用say方法的前提是say方法要么是b的私有属性，要么是b的公有属性，此时需要继承来实现，让B类继承A类

7.1 原型（链）继承

将子类的原型改写成父类的实例，把父类私有和公有属性都继承为子类公有的属性和方法

缺点：原型（链）继承要重新指定继承后的constructor属性

7.2 借用构造函数继承（call继承）

只继承父类的私有的属性和方法

在子类的构造函数中call执行父类的构造函数，此时父类是被当成一个普通函数使用的；把父类构造函数中的this修改为子类的实例

7.3 组合继承

把父类公有的继承到子类公有的，父类私有的继承到子类私有的

  • 将原型（链）继承和借用构造函数继承（call继承）组合到一起
    • 原型（链）继承：把父类私有和公有都变成子类公有的
    • 借用构造函数继承（call继承）：把父类私有的变成子类私有的

      注意（缺点）：子类会继承两次父类的私有属性；第一次来自于原型（链）继承时父类私有的成为子类公有的，另一次来自于借用构造函数继承（call继承）时把父类私有的变成了子类私有的。此时，子类的原型上有一份父类私有的，在子类私有的也有一份父类私有的

7.4 原型式继承

把父类的公有属性继承给子类的公有属性（只继承父类的公有属性、方法）

创建一个指定__proto__的对象：Object.create(protoObj) 返回一个新对象，这个新对象的__proto__指向protoObj

7.5 寄生组合式继承

把父类公有的继承为子类公有的，把父类私有的继承为子类私有的

  • 将原型式继承和借用构造函数继承（call继承）组合到一起
    • 原型式继承：继承父类公有的
    • 借用构造函数继承（call继承）：继承父类私有的

7.6 冒充对象继承

将父类私有、公有的属性都变成子类的私有属性

在子类的构造函数中创建一个父类的实例，遍历这个实例，把父类实例的属性、属性值都添加到子类实例身上

7.7 ES6 extends 关键字继承

class、extends继承：原理是寄生组合式继承

8. 面向对象-多态（重载、重写）

JS中没有真正的重载，因为JS中不允许声明多个同名函数，声明多个同名函数时，这个函数名就代表最后一个函数

重写：子类可以重写父类的属性和方法
重载：有多个同名函数，但是这些函数的参数（签名）不同；当传不同参数时，会自动调用不同函数执行；JS中没有真正的重载；但这个功能可以模拟，根据不同的参数做不同的事情
