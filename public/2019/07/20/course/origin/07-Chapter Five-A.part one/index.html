1. JS盒子模型属性

JS盒子模型：通过js的方法或者属性获取的元素对象的一系列描述盒子模型的属性；

内容的高度：是css样式设置的高度，和溢出不溢出没关系
真实的高度：盒子的实际的高度，包含溢出的部分

都是元素的一个属性，获取的值没有计量单位（Number类型）

1.1 client系列

只读属性，不能更改

clientWidth = 内容的宽度（width）+ 左右padding

clientHeight = 内容的高度（height）+ 上下padding

clientLeft = 元素的左边框（border）的宽度

clientTop = 元素的上边框（border）的高度

1.2 offset系列

只读属性，不能更改

offsetWidth = clientWidth + 左右边框（border）

offsetHeight = clientHeight + 上下边框（border）

1.3 scroll系列

window上有一个onscroll事件，其作用是监听滚动条的滚动事件

scrollWidth、scrollHeight只读属性，不能更改

scrollWidth

  • 1.没有溢出时，等于clientWidth
  • 2.溢出隐藏时，等于内容的真实宽度 + 左padding
  • 3.溢出滚动时，等于内容的真实宽度 + 左padding

当溢出时，就是真实宽度（clientWidth + 溢出部分【横向滚动条可以滚动的最大距离（即scrollLeft的最大值）】）

scrollHeight

  • 1.没有溢出时，等于clientHeight
  • 2.溢出隐藏时，等于内容的真实高度 + 上padding
  • 3.溢出滚动时，等于内容的真实高度 + 上下padding

当溢出时，就是真实高度（clientHeigth + 溢出部分【纵向滚动条可以滚动的最大距离（即scrollTop的最大值）】）

scrollLeft、scrollTop可读写属性，既能读取也能设置

scrollLeft = 横向滚动条卷起的距离（宽度）

scrollTop = 纵向的滚动条卷起的距离（高度）

1.4 偏移量

offsetParent

父级偏移参照物：距离当前元素最近的、有定位属性的父级元素；如果父级都没有定位属性，这个参照物就是body；但是body没有父级偏移参照物；

定位属性：position: relative（相对定位：正常位置） 、absolute（绝对定位：父级元素） 、fixed（绝对定位：浏览器窗口）、inherit（从父元素继承position属性的值）、static（默认值：正常出现，没有定位）

offsetLeft / offsetTop

当前元素相对于父级偏移参照物（offsetParent）内侧的偏移距离，left是左偏移，top上偏移；

获取某个元素距离body的上（左）偏移量时，并不能确定父级参照物一定是body；若父级参照物是body，直接获取元素的offsetLeft、offsetTop；若父级参照物不是body，需要让 (当前元素的offsetTop + (父级参照物的offsetTop + 父级参照物的clientTop)) ；若有多个父级参照物，需要把每一个父级参照物的偏移量和边框（border）都加在一起

4. 获取页面元素样式

opacity（透明度） : 0-1 IE低版本不能用
filter （IE的透明度）: alpha(opacity(0-100)) IE设置透明度

4.1 获取元素的行内样式

元素对象.style.xxx 只能获取或者设置元素的行内样式，拿不到CSS中的样式

4.2 获取元素所有计算生效的样式

  • getComputedStyle 方法；IE8及以下不兼容

    语法：window.getComputedStyle(元素对象, 伪类)

第二个参数不用伪类时用null

获取伪类的样式

  • CurrentStyle 属性；这个属性的值是一个对象，这个对象中包含了所有计算生效的样式 ； IE专属获取当前所有计算生效的样式

    语法：元素对象.CurrentStyle 【CurrentStyle是个属性，不能执行！】

    获取元素计算生效后的样式
  1. 标准浏览器： window.getComputedStyle(元素对象, 伪类【不需要伪类时写null】)；返回一个对象，对象中都是计算生效的样式，返回的对象是只读的，不能修改里面的样式；
  2. IE低版本：对象.currentStyle 属性，属性值是一个对象，对象存储的是这个元素计算生效后的样式

4.3 封装方法：获取元素计算生效后的样式

  • 计算生效后的样式：层叠样式表【css】和行内属性都包含；
  • 返回的对象是只读的，不能修改里面的样式
  • 为什么叫经过计算生效的样式？

    CSS样式根据权重(!important)、距离元素的远近（行内>内嵌>外链）会互相覆盖，最终生效的只有一个；

5. 图片懒加载（预加载、延时加载）- lazyLoad

常用的性能优化手段，类似的还有骨架屏

真实的项目中，如果页面中有很多的图片，一般不是页面打开时就加载全部图片，只加载用户可以看到的图片，用户暂时看不到的图片暂时不加载，等到某一个合适的时机（一般都是滚动出现/即将出现在浏览器的可视窗口内）再去加载这个图片

5.1 实现原理

img能展示成图片是因为img标签有一个src属性，src属性值是一个图片资源路径（url）。当浏览器解析HTML时，发现img有src属性，就会立刻按照资源路径去加载这张图片，如果这张图片真实有效，最后被下载成功，浏览器就会把它渲染成一张图片

5.2 图片懒加载原理

不直接给img设置src属性，没有src属性就不会去加载图片，通过一个自定义属性的方式，把图片的资源路径存储在img标签上（一般用data-src），等到合适的时机（一般都是滚动出现/即将出现在浏览器的可视窗口内），把提前存储的资源路径取出来（getAttribute），然后赋值给图片的src属性。此时img获得了自己的src，浏览器会立刻通过这个src去加载这个图片

5.3 单张图片延时加载

当图片进入浏览器的可视区域时再去加载这张图片

  • 如何计算图片什么时候进入可视区域？

    当（ 图片上外边缘距离body上内边缘的距离 - 页面滚动条卷起的距离 - 浏览器可是窗口的高度 <= 0 ） 时表示图片将进入可视窗口

  • 什么时候去计算

    当页面滚动条滚动时计算，onscroll事件：只要滚动条滚动，onscroll事件就会触发

  • 图片出现在可视区的临界值

    图片（上外边框距离body上内边框的距离【offsetTop】- 页面滚动条卷起的距离【scrollTop】- 浏览器可视窗口的高度【clientHeight】 = 0） 时，表示将要出现在浏览器的可视窗口中。一般为了优化用户体验，都是距离出现在可视窗口还剩一些（ >0 ）时就去加载。如果为了减小服务器压力，一般是让这个图片出现在可视区域一段距离（ <0 ）再去加载

5.4 多张图片延时加载

就是在页面中滚动时，把所有的图片都获取到，然后遍历这些图片，对每一个图片进行判断，判断其是否进入了浏览器可视窗口，如果进入了就去加载，如果没有就不加载

6. 瀑布流 - waterFall Flow

产品需求文档（PRD Production-Request-Description）：产品经理（或需求方）给的书面形式的文档，描述了应用的功能、交互细节等将来很可能只负责其中一部分
UI设计稿：psd、sketch
切图：把设计稿还原成HTML页面（写HTML+CSS）
UE（用户体验）文档：UE设计如何让用户体验更好

PRD：

实现一个项花瓣网一样的效果
1. 页面中有三列（*三个li浮动起来*），这些列的宽度相同（*设置宽度*），高度不固定（*不设置，靠内容撑开*）
2. 每一列分为若干块，这些块宽度相同（*width:100%*），高度不一定相同
3. 虽然这些块的高度不尽相同，但需要保证这些列最后的高度不能相差太大
4. 页面即将滚动到底的时候就去加载下一页（？怎么告诉服务器要下一页的数据 ？什么时候判断页面是否滚动到底 ？怎么判断页面滚动到底）
// 5. 页面图片需要懒加载【即便需求方不提，也要有性能优化的意识】（*多张图片的懒加载*）

宽度相同的多列不规则排布；这些列宽度相同，列中的每一块宽度相同，高度不同，但最终这些列的高度不能相差太大，当滚动条滚动到页面底部时去加载下一页；页面中所有的图片要延时加载

6.1 原理

  • 首先页面中的数据不能是写死的，是动态获取来的（Ajax），接着把数据绑定成HTML；在插入到列之前，先给这些列按照高度进行升序排序，在插入数据时先给高度最矮的插，再给第二矮的插，最后给最高的插，这样就能保证最后的高度相差不会太大
  • 页面滚动到底去加载下一页：什么时候计算？页面滚动时，即onscroll事件触发时
  • 怎么计算？页面的scrollHeight - 页面的scrollTop - 浏览器可是窗口的高度clientHeight <= 0 表示已经滚动到底了；所以就是在页面的onscroll实践中去计算剩余高度，当剩余高度为0时就表示到底了
