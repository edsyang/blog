webpack

webpack文档：https://webpack.js.org/
webpack中文文档：https://www.webpackjs.com

什么是webpack：

  • webpack是基于Node.js静态资源的模块打包器，当用webpack打包时，它会从一个起点开始查找各个模块依赖关系，并且按照这些依赖关系把这些文件（模块）打包成一个或者多个包

为什么使用webpack：

  1. webpack对模块化的支持十分强大，可以借助wekpack使用模块化开发；浏览器端的早期没有模块化，需要js文件只能通过script标签手动控制依赖管理这些js文件的引入顺序；而webpack可以通过模块化管理这些依赖
  2. 自动编译less、sass等css预处理语言
  3. 可以把基于js的扩展语言（如TypeScript）编译成js
  4. 开发时配置代理解决开发环境跨域问题（服务器转发）

22.2 配置npm scripts

前面通过命令行打包，需要配置一大堆参数；为了减少这种情况，我们需要配置npm scripts

  • 首先生成一个package.json 文件(npm init -y)
  • 在package.json 中有一个scripts对象，在这个对象中的每一个属性都是一个可以通过npm run执行的命令；

package.json

然后再cmd中执行npm run cusBuild就相当于在执行 webpack-cli –entry ./app/main.js –output ./public/bundle.js 命令；

在package.json的scripts下面的start自定义字段比较特殊，其他的自定义字段的执行需要写全npm run 自定义字段，而执行start字段只需要写npm start即可

22.3.3 配置source-map

控制source-map的路径：控制台 -> Settings -> Preference -> Sources -> Enable JavaScript source map

  • 代码打包之后，可读性变得非常差，不利于调试；而浏览器有一种socrce-map的机制，可以根据提供的source-map把打包之后的代码虚拟还原成打包之前的样子，方便开发调试
  • webpack在打包结束后可以自动生成source-map，只需要配置即可
  • source-map有很多种，source-map越详细，打包的速度就会越慢
  • source-map虚拟还原出来的js，可以像调试真实的js一样打断点（debugger）

    source-map 是浏览器的机制，可以把打包以后的文件虚拟还原成打包之前的样子，source-map越详细，打包的速度越慢；webpack可以生成source-map

22.3.5 配置webpack-dev-server

webpack-dev-server是同源策略中服务器代理的工具之一

个人理解：服务器代理和中间件的原理相似，都是通过该程序/机制来中转一些信息/文件等；只不过服务器代理是一个机制，而中间件是一个程序

在开发的过程中，webpack可以启动一个本地服务，主要有以下功能：

  1. 静态资源服务
  2. 让浏览器能够监听文件的变化，如果文件变化，页面自动刷新
  3. 可以代理客户端的请求(提供AJAX接口的代理)，解决开发环境的跨域问题
  • 1.安装webpack-dev-server
  • 2.在webpack配置中加入配置项

    配置详解
    contentBase: ‘./public’, 本地服务器所加载的页面所在的目录
    historyApiFallback: true, 单页面应用路由切换时不跳转
    inline: true, 实时刷新
    port: 8080, 启动时的端口号
    proxy: {} 代理配置

22.4 配置loader

webpack中一切皆模块，都可以import；如js、css、图片等，但原则上webpack只处理js文件，此时就需要借助loader处理其他类型的文件

loader：原则上，webpack只能处理js文件，但是通过使用不同的loader，webpack有能力调用外部的脚本或工具，实现对不同格式的文件的处理，比如说分析转换less为css，或者把下一代的JS文件（ES6，ES7)转换为现代浏览器兼容的JS文件，对React的开发而言，合适的Loaders可以把React的中用到的JSX文件转换为JS文件。

  • babel-loader（ES6/7转ES5 / JSX转ES）

  • css-loader/style-loader（处理css文件）

  • less-loader（处理less，将less文件转成css文件）

  • url-loader（处理图片）

  • …

    Loaders需要单独安装并且需要在webpack.config.js中的modules关键字下进行配置，Loaders的配置包括以下几方面：
- test：一个用以匹配loaders所处理文件的拓展名的正则表达式（必须）
- use：loader的名称（必须）
- include/exclude:手动添加必须处理的文件（文件夹）或屏蔽不需要处理的文件（文件夹）（可选）；
- query：为loaders提供额外的设置选项（可选）

22.5 插件（Plugins）

插件（Plugins）是用来拓展Webpack功能的，它们会在整个构建过程中生效，执行相关的任务。Webpack有很多内置插件，同时也有很多第三方插件，可以让我们完成更加丰富的功能。
