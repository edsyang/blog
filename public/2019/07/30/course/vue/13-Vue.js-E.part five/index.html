31. 动态组件&keep-alive

  • 有的时候，在不同组件之间进行动态切换是非常有用的，此时就需要用到动态组件；
  • 动态组件，需要使用内置的component组件
  • 在这个组件上有一个is属性，动态绑定该属性，当被绑定的值发生变化时，Vue会渲染is最新的值对应的组件；

动态组件是实现在多个组件间来回切换，用法；

  1. 写一个内置的component组件
  2. 给component动态绑定is属性，当is属性绑定的值发生变化时，Vue会按照is绑定的最新的值自动渲染对应的组件

    <component :is="title"></component> component组件会动态渲染is属性绑定的值所代表的组件；被切走的组件其实例会被销毁，存在里面的数据随之销毁，被切走的组件称为失活组件，而当前被渲染的组件称为激活组件

keep-alive
可以保持失活的组件不销毁

  • component虽然可以绑定动态组件，但是在组件切换时，原来的组件就会被销毁，数据会丢失，为了解决这个问题，我们使用keep-alive；
  • keep-alive也是内置的组件，它可以让失活的组件不销毁，所以保存在组件实例中的数据也不会销毁
  • 动态组件常和keep-alive一起使用

    包裹动态组件时，会缓存不活动的组件实例，而不是销毁它们。

32. 事件总线（EventBus）

  • 每个组件都是一个vue的实例，相互之间不能互通数据；
  • 要修改数据一定要通知，所以找一个第三方，让第三方监听事件，在事件触发时执行对应的修改数据的操作，这个第三方就是事件总线；

事件总线（EventBus）用来解决非父子组件之间通信的问题

用法：

  1. 创建一个事件总线（一个空的Vue实例）；let eventBus = new Vue()
  2. 谁的数据被修改，谁监听事件；在created钩子函数中，eventBus.$on(自定义事件名, 事件函数 - 它的形参接受$emit传来的数据)
  3. 谁发起修改，谁触发事件；eventBus.$emit(事件名, 数据)

    EventBus解决非父子组件通信问题：
    左侧是一个导航栏，上面有小气泡（99），右边是一个列表，每一行是一条数据，点击审核，审核通过后左侧小气泡中的数据要减小1；如果没有Vuex就要EventBus

33. 插槽（slot）

当引入组件时，可以向组件的自定义标签中嵌入内容，这些内容可以嵌入到子组件中，但是需要使用插槽，即<slot></slot>

如何使用slot

  1. 在子组件中提前定义插槽(占位)，即在需要的时候写一个<slot></slot>标签
  2. 把想要嵌入子组件的内容，写在子组件标签的里面

匿名slot和具名slot

  • 子组件中的slot标签可以设置一个name属性，设置了name属性的slot叫做具名slot，没有设置name属性的叫做匿名slot；
  • 如果子组件标签中的内容要添加到指定的slot，需要给这段模版指定slot属性
  • 子组件的标签中写了<div slot="before"></div>之后，这一段模版会添加到子组件的name属性为before的slot<slot name="before"></slot>

34. Vue的路由（vue-router）

  • 路由：根据不同的请求路径（pathname）执行不同的操作，路由是一个机制
  • 前端路由：单页面应用中由前端控制路由，根据不同的路由显示不同的页面（其实是不同的组件）；单页面应用（SPA）只有一个html文件，切换路由时是切换组件，而不会请求其他的html文件

Vue-router

  • Vue的单页面应用路由需要使用vue-router；vue-router监听页面的路由发生变化（页面的url就是前端路由），渲染对应的组件

    使用vue-router需要配置路由映射表

使用Vue-router需要使用以下组件

  • <router-link to="" tag=""></router-link> 是一个vue-router定义的标签，通过它可以切换到to属性指向的路由，如果有这个路由对应的组件，就会把这个组件展示到router-view；
  • <router-view></router-view> 用来展示路由对应的组件

    router-link叫做编程式导航，router-link是用来切换路由的，to属性是指定点击router-link时要跳转的路由，tag属性指定router-link以什么标签渲染到页面

使用Vue-router

  1. 安装并引入vue-router.js：npm install vue-router --save
  2. 配置路由映射表，即路由和组件的对应关系；单页面应用不会切换html，而是切换组件
  3. 创建VueRouter的实例，创建VueRouter实例时需要传入路由映射表
  4. 创建Vue实例时配置router属性，router属性的值是VueRouter的实例

    VueRouter 默认使用的hash模式，hash改变不会引起页面的刷新，但是会生成历史记录；

路由方法

除了router-link切换路由，vue-router还提供了对应的方法用于切换路由

常用的路由方法

配置router之后，才能使用$router对象

  1. this.$router.go(-1) 返回上一页（其实是组件）
  2. this.$router.push(router|router-config) 切换到指定的路由

嵌套路由

从当前页面可以进入到当前页面的子页面中，例如:

  • 当前页面的路由：/detail
  • profile 页面路由：/detail/profile
  • about 页面路由：/detail/about

    此时称profile和about是detail的嵌套路由或者是子路由；

如何实现子路由（路由嵌套）

  • 在/detail下面配置children属性;
  • 在/detail对应的组件模板中添加 router-view

路由参数

动态路由
vue-router的路由可以像 Node.js 的动态路由一样，可以配置动态路由；例如 /text/:id/:name 此时id和name就是动态的路由；而我们真正访问路由时，如 /text/12/13 此时的12和13就称为动态路由的参数；

需要研究的问题是如何获取动态路由，如id和name的值

获取动态路由的参数
this.$route.params 获取动态路由的参数
this.$route.query 获取问号传参的参数
