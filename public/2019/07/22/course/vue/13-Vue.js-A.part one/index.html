1. 初识Vue.js

  • 作者：尤雨溪 https://cn.Vuejs.org/v2/guide/
  • Vue.js是一个MVVM框架，框架思路是数据映射视图 / 数据驱动视图，有别于过往的DOM操作开发思想
  • 只需要关心数据，视图的更新由Vue.js处理

MVC和MVVM

  • MVC：M model层（数据层）；V view层（视图层）；C controller控制器，

    • MVC中数据是靠控制器渲染到视图中

  • MVVM：M model层（数据层）；V view层（视图层）；VM view-model层（视图模型层）

    • MVVM中是VM把视图和数据层联系起来，把数据映射成视图，当数据变更时会自动更新视图；
    • 双向数据绑定：把数据映射成视图，数据发生变化，更新视图；同时，如果视图发生变化，更新数据

Vue是响应式框架；所谓响应式不是css响应式，而是数据变化，视图自动变化

vue全家桶：vue + vue-router + vuex + vue-cli (webpack+vue)

2. 使用Vue.js

vm是vue的实例，在创建vue实例的时候，会把data里面的数据代理到vue实例身上

  1. 引入Vue.js
  2. 准备根DOM节点：Vue应用需要一个根DOM节点，所有用Vue写的东西，要放到一个元素里面
  3. 创建Vue的实例
  4. 通过Vue的模版语法（小胡子（moustache）语法：），把数据绑定到页面中

    学习框架的用法（背下来），按照步骤写代码

3. 双向数据绑定

  1. 第一个方向（数据映射视图）：通过Vue的模版语法，以及v-model指令把data里面的数据绑定到页面的html中
  2. 第二个方向：在视图（页面）中修改data中的属性的值，视图中所有依赖这个属性地方都会自动修改（更新）

双向数据绑定的原理

在Vue创建实例的时候，会遍历data中的属性，通过一个Object.defineProperty() 方法把data中的属性都改成getter和setter；当获取这个值时，会触发get方法，当修改属性值的时候会触发set方法；Vue在此前会收集所有依赖data属性的元素，当依赖的这个值发生改变时，会触发set方法，然后Vue会更新所有依赖这个数据的元素

Object.defineProperty() 定义对象属性的特性的

4. 向data中新增属性

创建实例以后增加/修改属性：使用vm.$set(vm.user, key, val)方法

  • key 要修改/新增的属性名
  • value 属性的新值

    用$set() 方法修改完数据，页面会自动更新

5. Vue的常用指令

指定：在Vue中，以v-开头的特殊的行内属性，Vue赋予了这些属性特殊的能力

  1. v-model 双向数据绑定 把表单元素的value和data中的属性绑定到一起，表单的value发生变化，data的属性也会跟着变化
  2. v-text 把data（不只是data）中的属性值绑定到DOM元素中，会覆盖元素中原有的内容；并且不能识别字符串中的html标签
  3. v-html 把data（不只是data）中的属性值绑定到DOM元素中；并且可以识别字符串中的html标签
  4. v-if / v-else 如果v-if绑定的值为true，就显示对应的元素；如果为false，显示v-else对应的元素；注意：v-else不是必选项，但必须结合v-if使用，而v-if可以单独使用
  5. v-show 如果值为true，对应的元素就会展示，如果为false，则隐藏；
  6. v-bind 绑定动态属性，动态绑定后就可以使用data中的该属性的值；v-bind可以简写为 :
  7. v-for 列表渲染
  8. v-on 事件绑定，可简写为@

只有表单元素才具有双向数据绑定的能力，所以只有表单元素才能使用双向数据绑定

v-show和v-if的区别：v-show是控制元素的display属性，而v-if是删除或者增加DOM节点实现DOM显示隐藏；尽量使用v-show，因为v-if操作DOM可能会导致回流重绘消耗性能

5.1 列表渲染 - v-for

v-for是一个指令，根据给定值生成多个相同元素，列表渲染

  • 要生成多个谁，就把v-for写谁身上
  1. v-for一个数组
  • item数组项，index当前项的索引
  • 在整个li标签和li子元素中都可以访问item和index的值
  1. v-for一个对象
  • val属性值，key属性名，obj是data中的对象
  • 对象有多少键值对就会生成多少个元素
  1. v-for一个数字
  • a是从1开始的一个数字，b相当于索引从0开始
  1. v-for一个字符串
  • a相当于字符串中每个字符，b是字符对应的索引

5.2 v-for练习

在Vue中，视图（html）是数据的映射，需要一个什么样的html结构，就需要有一个什么样子的数据结构

5.3 事件绑定 - v-on

Vue事件绑定，需要v-on指令，v-on:可以简写成@；后面跟不带on的事件名；无需获取DOM，直接在模版中绑定

  1. 在绑定事件时不传参，事件函数第一个参数是事件对象
  2. 如果绑定事件时传参了，传什么参数接收到的就是什么
  3. 如果既想用事件对象，又想传参，需要在绑定事件时，小括号内第一个写$event用来标识事件对象，后面再写真实的参数

5.4 事件修饰符

  • 事件修饰符：方便Vue中的事件处理，Vue中增加的一种特有的修饰符号
  • 用法：@事件名.修饰符 / v-on:事件名.修饰符

常见的事件修饰符：

  • 事件修饰符
    • .prevent 阻止元素的默认行为
    • .stop 阻止事件冒泡
    • .capture 事件在捕获阶段触发
    • .once 事件只会执行一次，执行完后会被移除
    • .self 自身的事件触发时才执行事件函数
  • 键盘事件（@keydown）修饰符：指定按下哪个按键时才触发事件函数
    • .enter 回车
    • .esc esc键
    • .delete 删除/退格
    • .space 空格
    • .tab tab键
    • .up 上
    • .right 右
    • .down 下
    • .left 左

事件修饰符可以连用，如：@click.prevent.stop 阻止元素的默认行为&事件冒泡

6. 练习：简易版todoList（事件）

Vue是数据驱动，不需要操作DOM，只需要操作数据，当Vue发现数据变化了，它会把页面中用到这个数据的所有的地方按照数据的最新值重新渲染（页面所有的变更都是基于数据的）

视图（html）是数据的映射，所谓映射是一个改另一个跟着改；当数据发生变更后，Vue会自动按照最新的数据渲染视图

PRD

  • todoList是待处理任务表，在input的框中输入一条任务，回车即可加入任务列表；在任务列表中每一行都有一个删除按钮，点击删除按钮即可从任务列表中删除该任务

7. 过滤器

用于处理数据，但是不会改变原数据的数据处理方式，一般用来格式化数据

  • 全局过滤器：Vue.filter(过滤器名, callback)；这种过滤器在任何地方都能使用
    • callback：(val) => {} -> 第一个参数（val）表示的是管道符前面的的值
  • 局部过滤器：写在Vue实例中的filters内的过滤器是局部过滤器；只能在当前组件可以使用
    • 过滤器可以连续使用，后面的过滤器的参数，是上一个过滤器的处理结果，数据会展示成最后一个过滤器的结果
    • 过滤器可以传参，参数是传给第二个形参的，第一个参数是管道符前面的值
  • | 叫做管道符，把它前面的值传递给过滤器函数的第一个参数；然后数据就会展示成过滤器函数的返回值

8. Vue - 表单元素

8.1 radio和v-model

多个radio（单选按钮） v-model同一个属性，这些radio就是一组，每次只能有一个选中，v-model的值是被选中的radio的value

表单元素不要轻易用@click

8.2 checkbox和v-model

  • 当checkbox（多选框）只有一个的时候，Vue会把val默认转为布尔值，如果转完之后是true，checkbox就是选中状态，否则不选中
  • 有多个checkbox要v-model一个数组，当选中某一个checkbox，这个checkbox的value会进到数组中，如果取消某一个checkbox，这个checkbox的value会从数组中移除

当checkbox选中状态发生变化时会出发onchange事件

8.3 select&textarea和v-model

  • select（下拉列表）下面的option都有一个value属性，select v-model的值和option的value相同的时候，option会被选中，当option切换的时候会触发onchange事件
  • textarea（文本域）通过v-model和某个属性双向数据绑定,和input是一样的

9. Vue - axios

在Vue中使用ajax请求数据，一般放到created钩子函数中

created是Vue生命周期的钩子函数，Vue里面绝大多数的ajax请求都是在这里发送的；在Vue的实例创建后会执行这个钩子；一般是把请求封装成一个方法放到methods里面，然后再created里面调用

10. bootstrap（css框架）

bootstrap是css框架，依靠类名来控制元素的样式；bootstrap内置了很多样式，都是有固定类名的，当需要这个样式时，在元素上加一个类名就可以了

第三方依赖包，需要下载，npm install bootstrap –save

  • bootstrap采用栅格布局，会把屏幕的宽分为12份
  • 常用的类名
    • .container 会让元素水平居中
    • .row 行，一行是12列
    • .col-lg-x 大屏幕
    • .col-md-x 中等屏幕
    • .col-sm-x 小屏幕
    • .col-xs-x 超小屏幕
