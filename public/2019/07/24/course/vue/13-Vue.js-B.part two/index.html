12. 计算属性 - computed

computed：计算属性，类似于过滤器，用于处理某个或者某些属性复杂展示逻辑，但是并不会改变原数据；目的就是不要在模版中写太多的逻辑

什么时候需要使用计算属性?

  1. 数据用来展示
  2. 需要展示的数据依赖其他数据，通过其他数据计算出来的
  3. 当它依赖的数据发生变化时，会重新计算

computed注意事项

  1. computed里面的属性会被vm所代理
  2. computed里面的属性和data/methods/filters等都不能重名
  3. computed的计算属性可以是一个函数，也可以是一个对象【对象中有set和get方法，取值执行get，设置执行set；而函数形式的计算属性只有get的情况，只能获取不能设置，如果设置会报错（Computed property ‘total’ was assigned to but it has no setter）】
  4. 如果一个值需要依赖其他属性计算而来，这时最好用计算属性

计算属性不能用在异步中

计算属性不能写在异步处理程序：ajax、定时器、promise的then…

13. 侦听器属性 - watch

watch: 侦听器属性，当需要监听某个属性的改变，且当其改变时需要做某些事情，此时就需要侦听器属性

能用表单的事件就用事件或者使用计算属性；这两种都不行的时候再用watch；

14. computed&watch的区别

computed 计算属性:

  1. 页面加载时就求值，当依赖的数据发生改变时会重新求值
  2. 不支持异步
  3. computed可以依赖多个属性，如果被依赖的属性有一个发生变化就会重新求值，相当于监控了多个属性

watch 计算属性:

  1. 页面加载时watch不执行，只有被监控的数据发生变化时才会执行对应的函数
  2. 支持异步
  3. 一个函数对应一个被监控的属性，只有当这个属性的值发生变化时才会执行

15. v-bind:class

v-bind 绑定动态属性
v-bind:class 动态绑定类名，实现动态操作样式

v-bind:class常见的形式

  1. 对象，{className1: 属性值1, className2: 属性值2…} 当属性值为true时，对应的className生效
  2. 三元运算符，三元运算符的返回值是生效的类名；如果条件为true，返回条件成立的值，否则返回条件不成立的值
  3. 方法调用，最终生效的类名是方法的返回值

16. v-bind:style

v-bind:style 动态绑定行内样式

v-bind:style常见的形式

  1. 一个对象，对象中都是样式，如：{color: ‘#000’, backgroundColor: ‘#fff’}
  2. 一个数组，数组项是样式对象，最终会把这些对象中的样式合并到一起
  3. 方法调用，在方法中返回一个样式对象

20. ES6模块化规范

模块化：在解决一些复杂的问题，按照一种分类的思想，把问题分解成若干个小模块功能，然后逐个实现这些小模块

模块化系统所具有的能力：

  1. 定义封装的模块
  2. 可以导出内容供其他模块使用
  3. 可以导入其他模块的内容

20.1 ESModule

  • CommonJS Node.js的模块化规范
  • AMD require.js 异步模块化规范

ESModule：ES6新增的模块化规范

  1. 一个js文件就是一个模块
  2. export 关键字 导出
  3. import 关键字 导入

    学习ESModule就是学习如何导出、如何导入

20.2 export导出

关键字，用于指定模块导出的内容

用法（推荐使用批量导出的方式，可以清晰的看出模块导出的内容）

  • 单个导出：export 变量声明语句
  • 批量导出：export { 要导出的变量 }
  • 默认导出：export default 要导出的变量

因为在导入时必须知道导出的时候名字都是什么，如果不知道的话没有办法导入；为了解决这个问题，ESModule提供了一个默认导出，所谓默认导出就是在其他模块在导入的时候可以不关心导出的名字是什么

一个模块的默认导出只能有一个，多了就会报错；如果模块是默认导出的，导入的时候不能写花括号，会报错；导入时的名字可以自定义

20.3 import导入

关键字，用于从其他模块中导入内容；导入几个都行

  • 导入语法：import { 变量 } from "带路径的模块文件"

  • 原名导出/导入：导出时变量叫什么，导入时还要叫什么

  • 在导入时给导出的变量重命名：需要使用as关键字；重命名后之前的名字就不能使用了 import { 原名字 as 新名字 } from 模块

  • 全部导入：把模块中导出的内容全部导入进来，放到名为变量名的对象 import * as 变量名（一个对象的变量名）from "带路径的模块文件"

  • 动态导入：import关键字只能用在同步导出，模块导入是静态的，发生在js代码执行之前，所以必须写在顶层/全局作用域中（不能写在函数或if中）；但是有一些场景需要动态导入模块，此时需要用import() 方法

    import() 方法支持动态导入，返回一个Promise实例，可以直接.then，then的第一个回调函数收到的参数就是模块中的内容；动态导入就是可以随时随地导入

    动态导入模块的场景：
    1.按需加载（导入）：如点击/滑动时去加载
    2.条件加载（导入）：满足某条件加载a模块，否则加载b模块
    3.模块路径是动态的：如路径是通过ajax从服务端获取的

    注意事项：

    1. import { 变量 } from “带路径的模块文件”
    2. 模块没有导出的变量，不能导入
    3. html在引入模块化的js文件时，script标签上要写type=”module”，否则报错；但在最新版的Chrome（75）浏览器上已经原生支持import，有进一步规范的趋势
    4. 写在import后面的花括号中的都是变量；自己声明的变量不能和花括号内的重名
